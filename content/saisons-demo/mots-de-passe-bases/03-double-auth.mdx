# La double authentification

Aussi appelee **2FA** (Two-Factor Authentication) ou **MFA**
(Multi-Factor Authentication). Principe : pour se connecter, il faut
**quelque chose qu'on sait** (le mot de passe) **ET quelque chose qu'on a**
(un telephone, une cle physique).

## Pourquoi c'est crucial

Microsoft a publie en 2019 une etude qui a fait date : la 2FA bloque
**99,9 %** des compromissions de compte. Meme si le mot de passe est
faible ou fuit, l'attaquant ne peut pas se connecter sans le second
facteur.

## Les 3 types de 2FA, du moins securise au plus securise

### 1. SMS - A EVITER
Code envoye par SMS. Vulnerable au "SIM swapping" (l'attaquant convainc
l'operateur de transferer le numero sur sa propre carte SIM). NIST
deconseille depuis 2017.

### 2. TOTP (app authenticator) - TRES RECOMMANDE
Une app sur le telephone (Authy, Google Authenticator, Microsoft
Authenticator) genere un code a 6 chiffres toutes les 30 secondes,
calcule localement avec une cle partagee a l'inscription. Pas de
reseau requis, pas de SIM swapping.

### 3. FIDO2 / Passkey - IDEAL
Cle physique (YubiKey) ou cle embarquee (Windows Hello, Touch ID,
Android). Impossible a hameconner : la cle ne fonctionne que sur le
vrai site, jamais sur un site de phishing.
