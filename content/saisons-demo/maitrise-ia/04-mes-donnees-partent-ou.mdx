debrief: |
  Tout ce que tu **tapes ou colles** dans ChatGPT, Claude grand public, Gemini gratuit, Copilot grand public **part dans les serveurs de l'editeur**, et **peut servir a entrainer leurs modeles** futurs.

  ## La hierarchie de confidentialite des LLMs

  | Niveau | Service | Politique par defaut |
  |---|---|---|
  | 🔴 Pire | ChatGPT gratuit / Gemini gratuit / Copilot gratuit | Prompts utilises pour entrainement, conservation 30+ jours |
  | 🟠 Moyen | ChatGPT Plus / Gemini Advanced / Copilot Pro | Pas d'entrainement par defaut, mais conservation longue |
  | 🟢 OK | ChatGPT Team / Enterprise / API + DPA | Engagement contractuel de non-entrainement, retention limitee |
  | 🔵 Souverain | Mistral (FR) en API, Anthropic UE | Donnees UE, conformite RGPD, audits possibles |
  | ⚫ Self-host | Ollama / Mistral local / LLM open weights | Aucune sortie de tes serveurs |

  ## Les 5 types de donnees a NE JAMAIS coller dans une IA grand public

  1. **Donnees clients identifiables** (noms, emails, telephones, contrats) - RGPD violation immediate
  2. **Donnees RH** (paies, evaluations, sante au travail) - secret medical + RGPD
  3. **Donnees financieres confidentielles** (M&A, levees, marges reelles)
  4. **Code source proprietaire** - le cas Samsung 2023 ou des ingenieurs ont fuite du code via ChatGPT
  5. **Strategie commerciale et propriete intellectuelle** - tarification, brevets en cours, contrats exclusifs

  ## Le cas Samsung 2023 (a documenter)

  Avril 2023 : des ingenieurs Samsung ont colle du code source confidentiel dans ChatGPT pour le debugger. Le code (incluant des secrets de fabrication de semi-conducteurs) est sorti des locaux de Samsung. Samsung a **interdit ChatGPT** pour ses 250 000 employes en mai 2023 et a developpe son LLM interne.

  Lecon : meme dans des entreprises ultra-soucieuses de la securite, **les employes utilisent ChatGPT sans realiser ce qu'ils envoient**. C'est le risque #1 du **shadow AI** (cf. episode 05).

  ## Les 3 reflexes a integrer

  1. **Avant de coller**, te poser la question : "Est-ce que je serais a l'aise si ce texte apparaissait demain sur un Forum public ?"
  2. **Anonymiser** systematiquement les noms, chiffres, identifiants. Remplacer "Acme Industries" par "Entreprise X", "3,2 M€" par "X M€". L'IA comprendra le contexte sans les details.
  3. **Utiliser le bon outil pour le bon usage** :
     - Brainstorming / reformulation banale : ChatGPT gratuit OK
     - Document confidentiel : Mistral Enterprise FR ou ChatGPT Enterprise + DPA signe
     - Code source proprietaire : LLM self-host (Ollama / Mistral local)

  ## Conformite : ce que dit le RGPD

  Le RGPD ne dit pas "interdit". Il dit "**ton sous-traitant doit etre encadre**" (article 28). Si tu utilises ChatGPT en pro :
  - Verifie qu'il existe un DPA signe entre ton entreprise et OpenAI (sur les plans Enterprise / API)
  - Inscris OpenAI dans ton **registre des sous-traitants** (art. 30)
  - Informe tes employes que les donnees envoyees sont traitees par un tiers US (transparence)

  Si tu utilises la version gratuite pour des donnees pro confidentielles, **tu es probablement en non-conformite RGPD** sans le savoir.

  > **Sources** : OpenAI Privacy Policy 2025, Samsung internal ChatGPT ban (Bloomberg, mai 2023), CNIL "Recommandations pour l'usage professionnel des IA generatives" (octobre 2024).
