# Verifier le domaine

Le **phishing** (hameconnage) est la porte d'entree de plus de 80 % des
cyberattaques en entreprise (source : Verizon DBIR 2024). L'attaquant
imite une marque de confiance pour soutirer un mot de passe ou de
l'argent.

## Les 4 trucages de domaine les plus courants

### 1. Homoglyphes
Remplacer une lettre par un caractere similaire :
- `microsoft.com` → `m1crosoft.com` (1 au lieu de i)
- `paypal.com` → `paypa1.com`
- `google.com` → `gooogle.com` (3 o)
- `humanix-cybersecurity.fr` → `humanix-cybersecunty.fr` (rt sans le i)

### 2. Sous-domaine trompeur
Le vrai domaine est a droite. Exemple :
- **microsoft.support-securite.com** : le vrai domaine est
  `support-securite.com`, PAS `microsoft.com`. C'est juste un sous-domaine
  nomme `microsoft`.
- Regle : lire de droite a gauche, le domaine est ce qui precede
  immediatement `.com` / `.fr` / `.org` etc.

### 3. TLD different
- `microsoft.com` (officiel) vs `microsoft.co` (suspect) vs `microsoft.support` (suspect)
- Si vous connaissez le domaine officiel (`.com`, `.fr`...), tout autre TLD est suspect.

### 4. Caracteres Unicode (IDN)
Domaines comme `аррlе.com` qui utilisent des lettres cyrilliques (а р р) visuellement
identiques aux latines (a p p). Les navigateurs modernes affichent un
warning, mais pas tous.

## Verifier en 10 secondes

1. **Survoler le lien** (sans cliquer) → l'URL reelle s'affiche en bas
2. **Lire l'URL de droite a gauche** pour identifier le vrai domaine
3. **Chercher les fautes** d'orthographe dans le domaine
4. **En cas de doute** : ne pas cliquer, ouvrir un nouvel onglet et taper
   l'URL officielle manuellement
