debrief: |
  Le **portail captif** (captive portal) est cette page qui s'ouvre
  automatiquement quand tu te connectes a un WiFi public (hotel,
  cafe, aeroport). C'est un mecanisme legitime d'authentification
  reseau, mais c'est aussi un vecteur de phishing classique.

  **Comment fonctionne un portail captif legitime :**

  1. Tu te connectes au WiFi (`Hilton-Guests`).
  2. Le routeur te donne une IP mais bloque tout trafic Internet.
  3. Quand tu tentes une URL HTTP, le routeur te redirige vers la page
     de login (le portail captif).
  4. Tu valides (CGU, login Hilton, code reseau).
  5. Le routeur autorise ton trafic Internet.

  **Pourquoi c'est piegeable :**

  - Le portail captif n'a PAS BESOIN d'etre HTTPS pour fonctionner -
    et beaucoup ne le sont pas.
  - Tu as l'habitude de voir ces pages s'ouvrir, donc tu y fais peu
    attention.
  - Les domaines varient selon les hotels (pas toujours hilton.com -
    parfois `wifi.hilton-paris.com`, `auth.cloud-wifi.eu`, etc.) - donc
    tu n'as pas de reference fixe.

  **Les 4 signaux d'alarme sur un portail captif :**

  **1. Domaine bizarre**

  - hilton.com, marriott.com = OK
  - hilton-secure-login.example.com = phishing
  - hotel-wifi-auth-server.tk = phishing
  - free-wifi.cn = phishing

  Reflexe : si le domaine ne te dit rien et que ce n'est pas le nom
  de l'hotel/entreprise telle qu'affichee a l'accueil, mefiance.

  **2. Demande de credentials sensibles**

  Un portail captif legitime demande :
  - Acceptation des CGU (case a cocher)
  - Email simple (pour newsletter / marketing)
  - Code reseau imprime sur le badge ou la facture

  Un portail captif legitime ne demande PAS :
  - Mot de passe d'un autre service (Google, Hilton Honors, banque)
  - Numero de carte bancaire pour "verifier le compte"
  - Numero de securite sociale

  Si on te demande un mot de passe d'un autre service ou un paiement
  pour un WiFi annonce gratuit : phishing.

  **3. Demande d'installation de certificat ou app**

  "Pour acceder au WiFi, veuillez installer ce certificat / cette
  application" - jamais legitime sur un WiFi public moderne. Si
  l'hotel veut absolument du HTTPS via leur portail, le certificat
  est deja signe par une autorite reconnue.

  Installer un certificat tiers = donner a l'attaquant la possibilite
  d'intercepter TOUT ton HTTPS futur.

  **4. URL HTTPS sans cadenas**

  Le portail captif HTTP est commun (acceptable). Mais s'il pretend
  etre HTTPS et que tu vois "Connexion non securisee" dans la barre
  d'URL, c'est un certificat invalide = mefiance maximale.

  **Defense ligne par ligne :**

  - Coche les CGU si demande, mais NE TAPE AUCUN PASSWORD existant
    (creer un mot de passe local jetable si necessaire).
  - Si on te demande un code WiFi, le mettre. Si on te demande
    Hilton Honors / Marriott Bonvoy / autre, REFUSER.
  - Pour aller vite : 4G/5G partage de connexion. Toujours.

  **Le piege "WiFi gratuit" en aeroport :**

  Beaucoup d'aeroports facturent le WiFi (15 EUR / heure typique).
  Un attaquant configure un AP "FreeAirportWiFi" gratuit. Tu te
  connectes pour eviter de payer. Le portail captif te demande email
  + mot de passe Google "pour verifier que tu es humain". Tu donnes.

  Defense : si le WiFi de l'aeroport est paye, c'est paye. Pas de
  shortcut "gratuit" sur un WiFi public dans un environnement non
  trust.
