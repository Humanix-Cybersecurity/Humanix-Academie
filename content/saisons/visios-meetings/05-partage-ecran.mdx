debrief: |
  Le **partage d'écran** en visio est l'une des sources de
  fuites involontaires les plus fréquentes. Quand tu partages,
  tu exposes :

  - **Tout l'écran** ou la fenêtre choisie
  - **Notifications** qui apparaissent
  - **Onglets** ouverts
  - **Documents** affiches par erreur
  - **Mots de passe** dans URLs / interfaces

  **Etudes (Microsoft 2023)** : 25% des employés ont au moins
  une fuite par mois lors de partages d'écran.

  **Les 6 types de fuites :**

  **1. Notifications**

  - **Email** : sujets sensibles
  - **Slack / Teams** : messages prives
  - **Signal / WhatsApp** : conversations perso
  - **Calendar** : prochains rendez-vous (revele)

  Solution : mode "ne pas deranger" / mode focus avant
  partage.

  **2. Autres fenêtres / onglets**

  - **Fenetre ouverte par erreur** (Cmd+Tab)
  - **Onglet Chrome avec données sensibles**
  - **Aperçu de fichier** sur Mac (preview rapide)

  Solution : partager **fenêtre seule**, pas tout l'écran.

  **3. Mots de passe / credentials**

  - **URL** avec token : `?api_key=abcdef`
  - **Console developpeur** ouverte
  - **Postman** avec credentials
  - **Gestionnaire mots de passe** ouvert

  Solution : vérification systematique + mode incognito si
  necessaire.

  **4. Données personnelles d'autres personnes**

  - **Liste contacts**
  - **Email reçu** d'une personne
  - **CV / candidature** d'un candidat
  - **Photos** dans les apps

  RGPD : violation possible.

  **5. Strategies / informations financieres**

  - **Spreadsheet** avec budget
  - **Roadmap produit**
  - **Dossier client**
  - **Rapport interne**

  **6. Bookmarks / historique navigation**

  - **Bookmarks** revele les sites fréquemment visites
  - **Historique** revele les recherches recentes
  - **Onglets sauvegardes**

  **Configuration anti-fuite avant partage :**

  **Étape 1 : Mode focus / ne pas deranger**

  **Mac** :
  - Centre de contrôle > Mode Focus > Activer
  - Bloque toutes les notifications

  **Windows 11** :
  - Notifications > Quick settings > Focus assist > Activer

  **iOS / Android** : mode "Ne pas deranger".

  **Étape 2 : Fermer les apps sensibles**

  Avant partage :
  - Fermer Slack / Teams / Signal / WhatsApp
  - Fermer email perso
  - Fermer Calendar (sauf si besoin)
  - Fermer gestionnaire mots de passe

  **Étape 3 : Preparer le contenu**

  - Ouvrir SEULEMENT les fichiers a partager
  - Fermer les autres onglets / documents
  - Vérifier ce qui est dans l'écran

  **Étape 4 : Choisir mode partage approprie**

  **Teams / Zoom / Meet** offrent plusieurs modes :

  **Partage 'fenêtre' (RECOMMANDE)** :
  - Une seule fenêtre choisie est partagee
  - Les notifications globales ne sont pas visibles (sauf
    dans la fenêtre)
  - **Limite l'exposition**

  **Partage 'tout l'écran' (RISQUE)** :
  - Tout est visible
  - Notifications, autres apps, etc.
  - **Éviter** sauf besoin spécifique

  **Partage 'navigateur'** :
  - Un onglet spécifique
  - Très limite

  **Partage 'Powerpoint mode présentation'** :
  - Mode dedie a la présentation
  - Cache notifications
  - **Ideal** pour présentations slides

  **Étape 5 : Vérification avant partage**

  - **Aperçu** de ce qui sera partage (Teams/Zoom/Meet le
    montrent)
  - **Vérification** : pas de surprise

  **Étape 6 : Surveillance pendant partage**

  - **Garde un oeil** sur ce qui apparait
  - **Si fuite** : arrêt immediat ("excusez-moi, un instant")
  - **Reprise** après correction

  **Le piege "je partage tout l'écran, c'est plus simple" :**

  Risque élève :
  - Notifications visibles
  - Apps visibles
  - Switching d'apps revele tout

  Solution : mode fenêtre + preparation.

  **Le piege "personne n'a vu, ça a passe vite" :**

  Faux :
  - 8-10 participants regardent attentivement
  - Capture / enregistrement possible
  - Quelqu'un toujours capture les "bonbons"

  Une fuite = potentiellement vue par tous.

  **Le piege "les notifications c'est pas grave" :**

  Le sujet de la notification suffit a fuir :
  - "Email de Apple : votre commande de XXX EUR est confirmee"
  - "Slack : Bob te répond sur #DG-secret-channel"
  - "Calendar : réunion suivante 14h - Negociation Acme"

  Mêmes les sujets revelent.

  **Le piege "j'ai un secret PWD qui apparait" :**

  Si un mot de passe / token / cle apparait :
  - **Action immediate** : changer le mot de passe / revoke
    le token
  - **Audit** : qui était dans la visio ?
  - **Notification RSSI** : peut être breach
  - **Documentation** de l'incident

  **Cas reels :**

  **Cas 1 - PME 2023 (notification licenciement)**

  Manager partage écran pour réunion d'équipe. Notification
  Slack : "Bob @PrivateChannel : Anne sera licenciee
  vendredi". Visible 5 sec dans la visio.

  Conséquences :
  - Anne (presente !) au courant
  - Climat équipe degrade
  - Procédure RH compliquee
  - Defi : qui a vu, qui a capture ?

  **Cas 2 - Developpeur 2024 (cle API leak)**

  Dev partage code dans Postman pendant demo. URL avec API
  key visible 2 sec. 8 participants externes (clients).

  Action :
  - Revocation immediate de l'API key
  - Génération nouvelle cle
  - Audit logs pour acces non-authorise

  Cout : 4h de revocation/regeneration + 1 mois de monitoring.

  **Cas 3 - DAF 2024 (virement urgent leak)**

  DAF partage spreadsheet avec budget. Notification email :
  "URGENT : virement 50K EUR en attente". Sujet visible.

  Risque : participants externes (commerciaux) connaissent
  un mouvement financier suspect. Pas de fuite reelle (pas
  d'utilisation), mais risque potentiel.

  **Cas 4 - Avocat 2023 (données client autre)**

  Avocat partage écran pour discuter d'un dossier client A.
  Onglet ouvert avec dossier client B (données nominatives).
  Visible 3 sec.

  Plainte client B (RGPD) :
  - 25K EUR amende CNIL
  - Procédure deontologique
  - Indemnisation client B

  **Configuration recommandee :**

  **Pour les pros qui partagent souvent :**

  - **Profil 'Visio'** dedie sur l'ordinateur (sans email
    perso, sans Slack, sans apps non-pro)
  - **Browser dedie** (autre que perso)
  - **Mode 'présentation'** active automatiquement quand
    Teams/Zoom partage

  **Pour les visios sensibles :**

  - **Pre-flight checklist** avant chaque visio
  - **Repetition** avec un collègue
  - **Browser incognito** pour éviter histo / cookies

  **Pour PowerPoint / Keynote :**

  - **Mode présentateur** : tu vois les notes, eux voient les
    slides
  - **Cache** automatiquement le reste du système
  - **Standard pour présentations longues**

  **Outils utiles :**

  **Microsoft PowerPoint Live (Teams)** :
  - Partage des slides directement (pas l'écran)
  - Tu peux naviguer librement
  - Participants voient seulement les slides

  **Zoom Whiteboard / Miro** :
  - Outils dedies pour brainstorming
  - Pas besoin de partager tout l'écran

  **OBS Studio** (gratuit) :
  - Composer une "scène" partagee
  - Cacher / montrer ce qu'on veut

  **Pour le RSSI / DSI :**

  Procédure post-fuite mots de passe :
  - **Revocation** immediate
  - **Audit** logs
  - **Information** affecte
  - **Sensibilisation** équipe

  **La règle d'or :**

  > **Avant de partager : mode focus / ne pas deranger +
  > fermer apps sensibles + choisir 'fenêtre seule' (pas
  > tout l'écran) + vérification preview. Pendant : surveiller.
  > Après fuite MDP : revocation immediate. PowerPoint Live /
  > mode présentateur = standard pro.**
