debrief: |
  Le **choix du cloud de sauvegarde** depend de plusieurs
  criteres : prix, souverainete, conformite, performance,
  fiabilite. Pas de solution universelle.

  **Les criteres de choix :**

  **1. Souverainete et juridiction**

  Cloud US :
  - **Cloud Act 2018** : les autorites US peuvent exiger
    l'acces aux donnees stockees, **meme** si datacenter en
    Europe
  - **CLOUD Act vs RGPD** : conflit potentiel
  - **Donnees soumises** : potentiellement transferable aux
    US sans consentement

  Cloud souverain FR / EU :
  - **Hebergement France / EU**
  - **Pas soumis au Cloud Act US**
  - **RGPD natif** (pas de transfert international)
  - **SecNumCloud** : qualification ANSSI = garantie ultime

  **Quand la souverainete est essentielle** :
  - **Sante** : donnees patients (HDS - Hebergement Donnees
    Sante)
  - **Defense / OIV** : Operateurs d'Importance Vitale
  - **Public** : administrations, collectivites
  - **NIS2 entites essentielles** : recommande
  - **Donnees personnelles sensibles** (categorie article 9
    RGPD)

  **Quand standard cloud OK** :
  - Donnees commerciales generales
  - Donnees techniques non-sensibles
  - PME standard sans contraintes specifiques

  **2. Prix**

  Comparatif cout pour 5 To stockes (sauvegarde mensuelle, peu
  d'extraction) :

  | Fournisseur | Stockage / mois | Extraction | Total/mois |
  |---|---|---|---|
  | **Backblaze B2** | 30 EUR | Gratuit (premiers 3x stockage) | 30 EUR |
  | **Wasabi** | 35 EUR | Gratuit | 35 EUR |
  | **AWS S3 Standard** | 100 EUR | 80 EUR / extraction 5 To | 180 EUR |
  | **AWS S3 Glacier** | 20 EUR | 100 EUR / extraction (lent) | 20 EUR |
  | **Azure Blob Cool** | 50 EUR | 60 EUR | 110 EUR |
  | **OVH Object Storage** | 50 EUR | Inclus 1x stockage | 50 EUR |
  | **Scaleway Object** | 40 EUR | Gratuit | 40 EUR |

  **Backblaze** et **Wasabi** sont les moins chers pour PME.
  **OVH** et **Scaleway** sont competitifs avec souverainete.

  **3. Conformite et certifications**

  | Fournisseur | RGPD | HDS (sante) | SecNumCloud | ISO 27001 | SOC 2 |
  |---|---|---|---|---|---|
  | **AWS** | OK | Possible (config) | Non | OUI | OUI |
  | **Azure** | OK | Possible | Non | OUI | OUI |
  | **Google Cloud** | OK | Possible | Non | OUI | OUI |
  | **OVH** | OK | OUI | OUI | OUI | OUI |
  | **Scaleway** | OK | OUI | En cours | OUI | OUI |
  | **Backblaze** | OK | Non | Non | OUI | OUI |
  | **Wasabi** | OK | Non | Non | OUI | OUI |
  | **Outscale** | OK | OUI | OUI | OUI | OUI |
  | **Cloud Temple** | OK | OUI | OUI | OUI | OUI |

  **SecNumCloud** = qualification ANSSI = standard le plus
  exigeant en France pour la cyber.

  **HDS** = obligatoire pour donnees de sante (etablissements
  hospitaliers, professionnels de sante).

  **4. Performance et fonctionnalites**

  - **API** : compatibilite S3 (standard universel)
  - **Versioning** : conservation des versions
  - **Object Lock** : immutable backup (cf ep 03)
  - **Encryption at rest** : chiffrement automatique
  - **Replication geo** : copies dans plusieurs datacenters
  - **Bande passante** : limites d'extraction

  **5. Resilience du fournisseur**

  - **Reputation** : nombre d'annees d'existence, base clients
  - **SLA** : disponibilite garantie (99.9% min)
  - **Plan B** : que se passe-t-il si le fournisseur ferme ?
    Migration possible ?

  **Recommandations selon profil :**

  **Profil 1 : PME standard (commercial, services)**

  - **Cloud principal** : Backblaze B2 ou Wasabi
  - **Cloud secondaire** : OVH Object Storage (souverainete
    bonus)
  - **Cout total** : 50-100 EUR/mois pour 5 To

  **Profil 2 : PME secteur sensible (sante, finance, public)**

  - **Cloud principal** : OVH (HDS) ou Outscale (SecNumCloud)
  - **Cloud secondaire** : Cloud Temple (souverain SecNumCloud)
  - **Cout total** : 100-300 EUR/mois pour 5 To

  **Profil 3 : Entite NIS2 essentielle / OIV**

  - **Cloud principal** : SecNumCloud obligatoire (Outscale,
    OVH SecNumCloud, Cloud Temple)
  - **Sauvegarde additionnelle** : on-premise air-gap (bandes)
  - **Cout total** : 500+ EUR/mois

  **Profil 4 : Particulier / TPE**

  - **Cloud principal** : OneDrive / Drive (souvent inclus
    M365 / Workspace)
  - **Cloud secondaire** : Backblaze Personal (5 EUR/mois) ou
    iDrive
  - **Cout total** : 5-20 EUR/mois

  **Le piege "AWS / Azure sont les meilleurs pour PME" :**

  Vrai pour les services applicatifs avances (ML, big data,
  serverless). Faux pour la SAUVEGARDE simple :

  - **Plus cher** : 3-5x plus cher que Backblaze pour
    sauvegarde
  - **Complexe** : pricing model alambique (storage class,
    extraction, requests)
  - **Pas souverain** : Cloud Act US

  Pour sauvegarde simple : Backblaze, Wasabi, OVH, Scaleway
  sont meilleurs choix.

  **Le piege "OVH = pas serieux car francais" :**

  Stereotype. La realite :
  - OVH est le **3e cloud mondial** par taille
  - **Multiples certifications** (HDS, SecNumCloud, ISO 27001)
  - **Datacenters** dans toute l'Europe
  - **Conformite RGPD** native

  Le seul reproche serieux : l'incendie de Strasbourg 2021
  (1 datacenter detruit). Lecons apprises, depuis : politique
  multi-zone obligatoire.

  **Le piege "le cloud souverain est plus cher" :**

  Faux. OVH et Scaleway sont **competitifs** avec les americains
  (parfois moins chers). La perception "americain = moins cher"
  vient :
  - Free tier AWS (gratuit pendant 1 an)
  - Marketing massif AWS / Azure
  - Mais sur le long terme, prix similaires.

  **Methodologie de selection :**

  **Etape 1 : Definir les criteres**

  - Volume de donnees (1 To, 10 To, 100 To)
  - Sensibilite (publique, interne, confidentiel, top secret)
  - Conformite requise (RGPD seul, HDS, SecNumCloud)
  - Budget (% du CA cyber)
  - Frequence d'extraction (rare, occasionnelle, frequente)

  **Etape 2 : Identifier 3-5 candidats**

  Selon les criteres, lister les options pertinentes. Eviter
  de comparer 1 candidat (pas de benchmark) ou 10+ (paralysie
  decisionnelle).

  **Etape 3 : POC (Proof of Concept)**

  - Tester 2-3 fournisseurs avec **comptes gratuits** /
    free tier
  - Sauvegarder un volume reduit (100 Go)
  - Tester la restauration (cf ep 02)
  - Mesurer les performances reelles

  **Etape 4 : Decision**

  - Documenter les criteres pondères
  - Choisir 1-2 fournisseurs
  - **Multi-cloud** : 1 principal + 1 secondaire (resilience)

  **Etape 5 : Implementation et audit**

  - Configuration (immutable, MFA, etc.)
  - Sauvegarde + tests
  - Audit annuel : performance, cout, conformite

  **Cas reels :**

  **Cas 1 - PME 2023 (migration AWS -> OVH)**

  PME 80 personnes utilisait AWS S3 pour les sauvegardes.
  Audit RSSI : 5K EUR/an de cout AWS, donnees clients
  sensibles (prospects EU). Migration vers OVH Object Storage
  : 2.5K EUR/an + souverainete.

  Resultat : 50% d'economies + conformite RGPD facilitee +
  argument commercial (clients europeens demandent souverainete).

  **Cas 2 - Hopital prive 2024 (HDS obligatoire)**

  Hopital teste les options pour sauvegarder dossiers
  patients. AWS / Azure : tres complique pour HDS (config
  specifique + couteux). OVH HDS : conformite native + 30%
  moins cher.

  **Choix** : OVH HDS + secondaire Outscale SecNumCloud.
  Conformite ARS + ANSSI assuree.

  **Cas 3 - PME US 2022 (incident ransomware sur S3)**

  Compte AWS compromis (keylogger sur PC admin). Attaquant a
  supprime les sauvegardes S3 (pas de Object Lock active).

  Cout : 200 K USD de paiement rancon + perte 4 jours.

  Lecon : le cloud, peu importe le fournisseur, doit etre
  configure correctement (Object Lock, MFA, audit log).

  **Liste des fournisseurs SecNumCloud (qualification ANSSI) :**

  Au 2026 :
  - **OVH** : OVHcloud SecNumCloud
  - **Outscale** : OUI Cloud
  - **Cloud Temple**
  - **Numerigo** (ex-Sopra Steria)
  - **Scaleway** : qualification en cours

  Ces fournisseurs sont **OBLIGATOIRES** pour :
  - OIV (Operateurs d'Importance Vitale)
  - Donnees classifees defense
  - Recommandes pour entites NIS2 essentielles

  **La regle d'or :**

  > **Pas de cloud universel. Selon contexte : PME standard
  > -> Backblaze/Wasabi (prix), PME sensible -> OVH/Scaleway
  > (souverainete), Entites essentielles -> SecNumCloud
  > obligatoire. Multi-cloud (2 fournisseurs) recommande pour
  > resilience.**
