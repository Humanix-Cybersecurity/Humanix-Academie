debrief: |
  La **règle 3-2-1** est le standard universel des sauvegardes
  professionnelles. Etablie dans les années 2000 par les pros
  IT, encore d'actualite et **renforcee** par les attaques
  ransomware modernes.

  **La règle 3-2-1 :**

  - **3** copies au total des données
  - **2** supports différents
  - **1** copie off-site (hors du site physique)

  **Variante moderne 3-2-1-1-0** (anti-ransomware) :

  - **3** copies
  - **2** supports différents
  - **1** off-site
  - **1** copie immutable (read-only, anti-ransomware)
  - **0** erreur lors du test de restauration

  **Pourquoi la règle 3-2-1 ?**

  **3 copies** :
  - 1 copie = les données originales (production)
  - 2 copies = couvre la panne d'1 sauvegarde
  - 3 copies = couvre la corruption / perte de 2

  **2 supports différents** :
  - HDD + SSD
  - Disque local + cloud
  - Bande + disque
  - **Diversification du risque** : si un type de support
    echoue (firmware bug, panne), l'autre reste

  **1 off-site** :
  - **Anti-ransomware** : si l'attaquant chiffre tout sur
    place, l'off-site est intact
  - **Anti-incendie / inondation** : sinistre du site
    n'affecte pas l'off-site
  - **Anti-vol** : vol des matériels du site = off-site
    intact

  **Implementation pour PME (80 personnes) :**

  Total coût : **40-150 EUR/mois** pour PME 80 personnes (1-5
  To).

  - Anti-ransomware option (Object Lock)

  - Hebergement France (RGPD compliance native)
  - Pour secteurs sensibles

  **Le piege "le sync c'est une sauvegarde" :**

  FAUX. La synchronisation (OneDrive, Dropbox, iCloud) **n'est
  PAS une sauvegarde** :

  - Si tu supprimes un fichier, il est supprime partout
  - Si un ransomware chiffre un fichier, il est chiffre partout
  - Si tu modifies un fichier par erreur, la modification se
    propage

  Solution :
  - Active la **conservation des versions** (OneDrive, Dropbox)
    : tu peux restaurer une version anterieure
  - **Sauvegarde additionnelle** vers un cloud différent
  - **Sauvegarde immutable** ideale (cf ep 03)

  **Le piege "1 sauvegarde sur disque externe = OK" :**

  Risques avec disque externe seul :
  - **Vol** : disque externe vole avec le PC
  - **Incendie / inondation** : disque externe a côté du PC
  - **Panne** : disque externe peut tomber en panne (10-20%
    sur 5 ans)
  - **Ransomware** : si connecte au PC pendant l'attaque,
    chiffre

  Solution : disque externe = OK pour la "copie 2 sur support
  différent", mais besoin d'une copie 3 off-site (cloud).

  **Le piege "off-site = au bureau d'a côté" :**

  PRESQUE faux. Off-site = **localisation physique distincte**
  pour éviter :
  - Incendie / inondation de tout le bâtiment
  - Vol massif (cambriolage)
  - Coupure electrique générale

  "Off-site" = ideallement différent ville / region. Cloud
  d'un fournisseur tiers = parfait off-site.

  **Frequence de sauvegarde :**

  **Selon criticite des données** :

  | Criticite | RPO max | Frequence |
  |---|---|---|
  | **Critique** (transactions, paie) | 1h | Continue (replication) ou horaire |
  | **Importante** (clients, projets) | 24h | Quotidienne |
  | **Standard** (documents, archives) | 7 jours | Hebdomadaire |
  | **Faible** (archives anciennes) | 30 jours | Mensuelle |

  **RPO** (Recovery Point Objective) = durée max de perte de
  données toleree.

  **Versioning et retention :**

  Les sauvegardes doivent garder plusieurs **versions** dans le
  temps :

  **Politique GFS (Grandfather-Father-Son)** :
  - **Daily** : 7 jours (Son)
  - **Weekly** : 4 semaines (Father)
  - **Monthly** : 12 mois (Grandfather)

  Permet de :
  - Restaurer une version recente
  - Restaurer une version ancienne (utile si corruption non
    detectee depuis longtemps)

  **Standard ANSSI / NIST :**

  - 7 jours daily
  - 4 weeks weekly
  - 12 months monthly
  - **Optionnel** : 1 archive annuelle 5+ ans

  **Outils gratuits / OSS pour PME :**

  **a) Veeam Backup Free**
  - Gratuit jusqu'à quelques machines
  - Standard PME
  - Backup VMware / Hyper-V / physique

  **b) Restic**
  - OSS gratuit
  - Cloud-native (S3, B2, Azure, GCS)
  - Chiffrement bout-en-bout
  - Deduplication efficace

  **c) Borg Backup**
  - OSS gratuit
  - Très efficace deduplication
  - Linux / Mac

  **d) Duplicati**
  - OSS gratuit, GUI
  - Multi-OS
  - S3 / B2 / OneDrive / Drive

  **Cas reels :**

  **Cas 1 - Hôpital français 2014 (inondation)**

  Hôpital avec sauvegarde sur NAS dans la salle serveur du
  sous-sol. Inondation due a une rupture de plomberie. Tous
  les serveurs et le NAS detruits. **Aucune copie off-site**.

  Résultats :
  - Perte de 6 mois de dossiers patients
  - 18h d'indisponibilite
  - Cout estime : 800 K EUR
  - Reputation : article presse + plainte de patients

  Lecon : off-site obligatoire.

  **Cas 2 - PME ransomware 2023 (sauvegardes chiffrees)**

  PME avec sauvegarde sur NAS connecte au réseau. Ransomware
  s'est propage du PC infecte vers le NAS et a chiffre **les
  sauvegardes**. Pas de copie off-site.

  Résultat : paiement de 80 K EUR de rançon (ANSSI deconseille
  fortement).

  Lecon : sauvegardes immuables ou off-site indispensables.

  **Cas 3 - PME 2024 (succes par 3-2-1)**

  PME 60 personnes, attaque ransomware. Sauvegardes :
  - Production
  - NAS local (chiffre par ransomware)
  - **Cloud Backblaze** (off-site, immutable)

  Restauration depuis Backblaze en 18h. **Pas de paiement de
  rançon.** Cout total : 5K EUR (perte d'activite 18h) +
  surcout ransomware 0 EUR.

  ROI de la règle 3-2-1 : évident.

  **La règle d'or :**

  > **3 copies, 2 supports, 1 off-site. Pas de négociation. Le
  > sync N'est PAS une sauvegarde. Le 'même site' n'est PAS
  > du off-site. Cout marginal : 30-100 EUR/mois pour PME. ROI
  > face a un ransomware : > 1000x.**
