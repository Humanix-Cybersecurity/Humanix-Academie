debrief: |
  Le **ransomware moderne** ne se contente plus de chiffrer
  les serveurs. Il cible **specifiquement les sauvegardes**
  pour empecher la restauration et forcer le paiement.

  Etude Sophos 2023 : **80% des attaques ransomware** ciblent
  les sauvegardes en premier.

  **Comment le ransomware compromet les sauvegardes :**

  **Methode 1 : Sauvegarde sur disque connecte**

  - NAS connecte au reseau via SMB/NFS
  - Disque externe USB connecte au PC infecte
  - Le ransomware se propage et chiffre TOUT ce qui est
    accessible

  Defense : immutable backup ou disque deconnecte ("air-gap")

  **Methode 2 : Compromission du compte cloud**

  - Keylogger sur PC du DSI
  - Phishing du DSI
  - Mot de passe reutilise (fuite externe)
  - L'attaquant accede a la console cloud (AWS, Azure,
    Backblaze, Google)
  - Suppression manuelle des sauvegardes

  Defense : MFA + immutable backup + segmentation acces

  **Methode 3 : Attaque sur le software de sauvegarde**

  - Veeam, Acronis ont eu des failles 0-day exploitees
  - L'attaquant prend le controle du serveur de sauvegarde
  - Suppression / modification des sauvegardes

  Defense : patches, segmentation, MFA

  **Methode 4 : Insider rancunier**

  - Employe IT qui veut nuire
  - Compte admin actif
  - Suppression des sauvegardes avant le sinistre

  Defense : immutable + audit + separation des taches

  **L'immutable backup : la defense ultime**

  **Definition** : sauvegarde **non modifiable et non
  supprimable** pendant une duree definie, MEME par un
  administrateur compromis.

  Technologie typique : **WORM** (Write Once Read Many) -
  l'objet est ecrit une fois, ne peut etre modifie.

  **Comment ca marche :**

  - Tu sauvegardes le fichier X avec retention 30 jours
  - Pendant 30 jours :
    - Le fichier ne peut PAS etre modifie
    - Le fichier ne peut PAS etre supprime
    - Meme avec credentials admin
    - Meme avec attaque API
  - Apres 30 jours : le fichier expire et peut etre
    supprime / archive

  **Implementations principales :**

  **a) Object Lock (S3 et compatibles)**

  - **AWS S3** : Object Lock natif depuis 2018
  - **Backblaze B2** : Object Lock disponible
  - **Wasabi** : Compliance Mode = immutable
  - **Cloudflare R2** : a venir
  - **OVH Object Storage** : disponible

  **2 modes** :
  - **Compliance Mode** : aucune suppression possible (meme
    par root) jusqu'a expiration
  - **Governance Mode** : suppression possible avec
    permissions speciales

  Pour anti-ransomware : **Compliance Mode** OBLIGATOIRE.

  **b) NAS avec immutable snapshots**

  - **Synology** : Snapshot Replication avec verrouillage
  - **QNAP** : Snapshot Vault
  - **TrueNAS** : ZFS snapshots immutables

  Le NAS lui-meme protege les snapshots de la suppression.

  **c) Bandes magnetiques offline**

  - LTO-8 / LTO-9 (12-18 To par bande)
  - Stockees physiquement separees du systeme
  - **Air-gap** absolu : pas connecte = pas accessible
  - Solution preferee des grandes entreprises (banques)

  **d) Veeam Hardened Repository**

  - Linux server immutable (XFS) configure specifiquement
  - Veeam V11+ supporte
  - Solution PME / ETI

  **Configuration Backblaze Object Lock (exemple) :**

  ```
  Bucket: my-immutable-backups
  Object Lock: Enabled
  Default retention mode: Compliance
  Default retention period: 30 days
  ```

  Une fois configure, **MEME** :
  - L'admin Backblaze (cote utilisateur)
  - Backblaze support
  - Une attaque qui prend le controle complete

  ne peut **PAS** supprimer les sauvegardes pendant 30 jours.

  **Combine avec un service de sauvegarde** :
  - **Restic** + Object Lock = sauvegarde immutable
  - **Duplicacy** + Object Lock
  - **Veeam** + S3 Object Lock

  **Politique de retention recommandee :**

  | Type | Retention immutable | Retention totale |
  |---|---|---|
  | Daily | 7-30 jours | 30 jours |
  | Weekly | 4 semaines | 90 jours |
  | Monthly | 12 mois | 12 mois |
  | Yearly archive | 5 ans | 7 ans |

  La **retention immutable** est la duree pendant laquelle
  AUCUNE suppression n'est possible. La **retention totale**
  inclut la possibilite de purge apres expiration.

  **Le piege "le NAS Synology a des snapshots, c'est OK" :**

  Les snapshots Synology par defaut peuvent etre **supprimes
  par admin**. Pour immutable :

  - Activer **Snapshot Replication** avec lock
  - **Mot de passe** different du compte admin principal
  - **MFA** sur le compte
  - **Retention** definie (impossible de raccourcir)

  Sans configuration immutable, les snapshots ne sont PAS
  protections suffisantes contre ransomware.

  **Le piege "le cloud est intrinsequement immutable" :**

  FAUX. Le cloud par defaut permet la suppression. Il faut
  **explicitement** activer :
  - Object Lock (S3, B2, Wasabi)
  - WORM (Azure, GCS)
  - Versioning + MFA delete (AWS S3)

  Sans configuration : compte compromis = sauvegardes
  supprimees.

  **Le piege "MFA suffit, pas besoin d'immutable" :**

  MFA est essentiel, mais pas suffisant :
  - **Bypass MFA** : SIM swap, social engineering, malware
    qui vole le token
  - **API key** : peut bypass MFA dans certains cas
  - **Insider** : si l'admin lui-meme veut nuire

  Solution : MFA + Object Lock + audit log = defense en
  profondeur.

  **Cas reels :**

  **Cas 1 - PME US 2023 (succes par immutable)**

  PME 50 employes attaquee par ransomware Conti. Le ransomware
  a :
  - Chiffre tous les serveurs
  - Chiffre le NAS local
  - **Tente** de supprimer les sauvegardes Backblaze

  Mais : Object Lock activee. Les sauvegardes restent intactes.

  Restauration en 24h depuis Backblaze. **Pas de paiement.**
  Cout : 10 K USD (perte d'activite).

  **Cas 2 - Hopital US 2022 (echec sans immutable)**

  Sauvegardes Veeam stockees sur serveur dedies, pas
  d'immutable. Ransomware compromis le compte admin Veeam,
  supprimes toutes les sauvegardes avant chiffrement local.

  **600 K USD** de rancon paye + 6 semaines de degradation +
  poursuites legales.

  **Cas 3 - Banque francaise 2024 (defense en profondeur)**

  Banque avec triple defense :
  - Sauvegardes Veeam locale
  - Sauvegardes cloud avec Object Lock (AWS S3)
  - Sauvegardes bandes LTO-8 dans coffre-fort

  Attaque ransomware : tout chiffre sauf bandes LTO. Restauration
  depuis bandes en 72h.

  **Cout total minimal**, conformite assurance cyber maintenue.

  **Recommandations selon taille :**

  **TPE (5-20 personnes)** :
  - 1 sauvegarde locale (NAS)
  - 1 sauvegarde cloud avec Object Lock (Backblaze B2)
  - **Cout** : 30-50 EUR/mois

  **PME (20-100 personnes)** :
  - 1 sauvegarde locale (NAS Synology avec snapshots
    immutables)
  - 1 sauvegarde cloud avec Object Lock (Backblaze ou Wasabi)
  - **Cout** : 80-200 EUR/mois

  **ETI (100-500 personnes)** :
  - 1 sauvegarde locale (Veeam Hardened Repository Linux)
  - 1 sauvegarde cloud Object Lock (S3)
  - 1 sauvegarde bande LTO offline (optionnel)
  - **Cout** : 500-2000 EUR/mois

  **Grande entreprise (500+)** :
  - Multi-cloud + bandes + Veeam + air-gap dedie
  - **Cout** : 2000+ EUR/mois

  **NIS2 et immutable :**

  L'article 21 mesure 3 (continuite des activites) impose des
  sauvegardes resilientes au ransomware. Les implementations
  conformes incluent obligatoirement :

  - **Au moins 1 copie immutable** ou offline
  - **Tests de restauration** documentes (cf ep 02)
  - **Politique de retention** claire
  - **Anti-corruption** verifiable

  **Outils gratuits / OSS :**

  - **Restic** + S3/B2 Object Lock = sauvegarde immutable
    gratuite
  - **Veeam Backup Free** + Object Lock storage
  - **Duplicati** + Object Lock storage
  - **Borgmatic** + B2 immutable

  **Configuration minimale pour PME :**

  ```bash
  # Restic vers Backblaze B2 avec Object Lock
  export B2_ACCOUNT_ID=xxx
  export B2_ACCOUNT_KEY=xxx
  
  restic -r b2:my-immutable-bucket:/backups init
  restic -r b2:my-immutable-bucket:/backups backup /data
  
  # Backblaze configuration cote console:
  # Bucket > Settings > Object Lock > Enable
  # Default retention: 30 days
  # Mode: Compliance
  ```

  **La regle d'or :**

  > **80% des ransomware ciblent les sauvegardes. Object Lock
  > / immutable backup = defense ultime. Meme avec compte
  > compromis, suppression impossible. Cout marginal : 0-50%
  > de plus que sauvegarde standard. Standard NIS2.**
