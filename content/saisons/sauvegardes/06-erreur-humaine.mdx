debrief: |
  L'**erreur humaine** est responsable de **40-60% des pertes
  de données** en entreprise (statistique Gartner / ESG). Plus
  que les ransomware ou les pannes hardware.

  Causes typiques :
  - Suppression accidentelle (dossier, fichier, ligne)
  - Modification non-voulue (formules ecrasees, données
    remplacees)
  - Mauvaise destination (envoi a la mauvaise personne)
  - Mauvais filtre (export incomplet)

  **La bonne nouvelle** : la plupart des plateformes modernes
  ont **plusieurs niveaux** de protection contre l'erreur
  humaine.

  **Les niveaux de restauration disponibles :**

  **Niveau 1 : Corbeille (recycle bin) - 30 jours**

  La première ligne de defense. Le fichier est dans la
  corbeille, pas vraiment supprime.

  **SharePoint / OneDrive** :
  - Site Recycle Bin : 93 jours par defaut
  - Site Collection Recycle Bin : second-stage recycle bin

  **Google Drive** :
  - Trash : 30 jours
  - Restoration en 1 clic

  **Dropbox** :
  - Deleted files : 30 jours (Plus), 180 jours (Business)

  **Action** : vérifier la corbeille IMMEDIATEMENT. Souvent
  le fichier y est encore.

  **Niveau 2 : Conservation des versions - 180 jours**

  Si le fichier a été modifie / corrompu (pas supprime), tu
  peux revenir a une version précédente.

  **Microsoft 365** :
  - **Versioning** : jusqu'à 500 versions / fichier
  - Tab "Version History" dans Office

  **Google Workspace** :
  - Versioning automatique
  - "File > Version history > See version history"

  **Action** : si fichier modifie par erreur, revert a
  version précédente.

  **Niveau 3 : Sauvegarde Microsoft / Google - 60-90 jours**

  Au-dela des corbeilles utilisateur, Microsoft et Google ont
  des sauvegardes au niveau plate-forme.

  **Microsoft 365** :
  - SharePoint : sauvegarde 60 jours après deletion définitive
  - **Restoration via support Microsoft** (si dans la
    fenêtre)

  **Google Workspace** :
  - Vault (E5) : conservation prolongee
  - Restoration via support Google si dans 25 jours
    additionnels

  **Action** : si non-trouve dans corbeille, contacter le
  support.

  **Niveau 4 : Sauvegarde tiers - jusqu'à 7 ans**

  Solutions tierces de sauvegarde (indépendantes de Microsoft /
  Google) :

  - **Veeam Backup for Microsoft 365**
  - **Acronis Cloud Backup**
  - **Druva inSync**
  - **Spanning** (Box, Microsoft, Google, Salesforce)

  **Politique de retention typique** :
  - Daily : 30 jours
  - Weekly : 90 jours
  - Monthly : 1 an
  - Yearly : 7 ans

  **Action** : si tout le reste a echoue, restoration via
  l'outil de sauvegarde.

  **Niveau 5 : Disaster recovery - durée de retention**

  Sauvegarde longue durée (cf ep 04 - cloud backup). Pour les
  archives critiques.

  **Procédure ideale après erreur humaine :**

  **Heure 0 - 5 min : Action immediate**

  1. **NE PAS** continuer a manipuler les fichiers (éviter
    aggravation)
  2. **Vérifier la corbeille** rapidement
  3. **Appeler le DSI** / hotline IT
  4. **Documenter** : que s'est-il passe, quand, quels
    fichiers

  **Heure 0 - 30 min : Investigation**

  1. **Audit log** SharePoint : qui a fait quoi quand
  2. **Identifier exactement** ce qui a été supprime
  3. **Vérifier les versions** disponibles

  **Heure 30 min - 2h : Restauration**

  1. **Restoration de la corbeille** si possible
  2. **Sinon** : restoration depuis sauvegarde tiers
  3. **Sinon** : support Microsoft / Google
  4. **Vérification** : tout est-il revenu ?

  **Heure 2h - 24h : Validation**

  1. **Test** des fichiers restaures (ouvrabilite, integrite)
  2. **Communication** aux utilisateurs (peuvent-ils
    travailler ?)
  3. **Documentation** de l'incident

  **Après : Lessons learned**

  1. **Pourquoi** est-ce arrive ?
  2. **Comment** prévenir la recidive ?
  3. **Mise a jour** des procédures / formation

  **Le droit a l'erreur :**

  Une bonne politique d'entreprise inclut le **droit a
  l'erreur** :

  - **Pas de sanction** disciplinaire pour erreur honnête
  - **Encouragement** a signaler immédiatement
  - **Climat** de confiance pour ne pas cacher

  Pourquoi : si l'employé craint la sanction, il :
  - Cache l'erreur
  - Tente de "reparer" lui-même (souvent en aggravant)
  - Le délai augmente, la restauration devient impossible

  **A l'inverse** : sanctions seulement pour :
  - Negligence repetee (après formation)
  - Faute volontaire / malveillance
  - Cachement deliberé après erreur

  **Prévention de l'erreur humaine :**

  **1. Formation**

  - Formation initiale aux outils
  - Formation reguliere (1x/an)
  - Tutoriels accessibles

  **2. Permissions appropriees**

  - **Principe du moindre privilège** : tu ne peux supprimer
    que ce que tu dois pouvoir supprimer
  - **Lecture seule** par defaut pour la plupart des
    utilisateurs
  - **Permissions admin** : peu d'utilisateurs

  **3. Confirmation / Double-check**

  - **Pop-up de confirmation** pour suppressions
  - **Mode "soft delete"** par defaut (pas de suppression
    définitive immediate)
  - **Validation par un tiers** pour actions critiques

  **4. Backup automatique**

  - **Sauvegarde automatique** transparente pour l'utilisateur
  - **Versioning** auto active

  **5. UX claire**

  - Boutons "Supprimer" en couleur (rouge) pour attention
  - **Loin** du bouton "Sauvegarder" (éviter mauvais clic)
  - Pas de raccourci clavier pour suppression définitive

  **Configuration SharePoint anti-erreur :**

  Dans SharePoint admin center :

  - **Versioning** : 500 versions max par fichier
  - **Recycle bin retention** : 93 jours (max)
  - **Soft delete retention** : 30 jours
  - **Audit log** : conservation 180 jours min

  **Plus** :
  - **Sauvegarde tiers** (Veeam, Acronis) avec retention 1 an
  - **Sensitivity labels** : protection des fichiers
    confidentiels
  - **DLP** : prévention de partage non-voulu

  **Cas reels :**

  **Cas 1 - PME 2024 (succes par signalement rapide)**

  Comptable supprime accidentellement le fichier paie
  (sauvegarde le mardi). Action immediate :
  - 10 min après : appel DSI
  - 30 min après : restauration depuis corbeille SharePoint
  - 1h après : tout est OK

  Cout : 0 EUR + 1h de reaction. Aucune sanction (droit a
  l'erreur).

  **Cas 2 - PME 2023 (echec par cachement)**

  Manager supprime des données client et tente de "reparer"
  pendant 3 jours sans signaler. Au final, l'erreur est
  decouverte par un autre employé.

  - 3 jours = depasse la corbeille
  - Sauvegarde tiers : oui mais incomplete
  - Restauration partielle = 30% des données perdues

  Cout : 50 K EUR (données client) + sanction disciplinaire
  pour cachement (pas pour erreur).

  **Cas 3 - Cabinet conseil 2022 (formule Excel ecrasee)**

  Junior remplace par erreur des formules par des valeurs
  fixes dans un fichier Excel client. Decouverte 2 semaines
  plus tard.

  - **Versioning Microsoft** : 30 versions disponibles
  - Restoration de la version 14 jours auparavant
  - Reapplication des modifications recentes manuelles

  Cout : 4h de re-travail. Pas de perte de données client.

  **Statistiques importantes :**

  - **40-60% des pertes** sont dues a erreur humaine (Gartner)
  - **80% des erreurs** sont detectees < 24h SI signalees
    immédiatement
  - **Restauration** depuis corbeille : 95% réussite si
    < 30 jours
  - **Cachement** : multiplie par 10 le coût final

  **Outils gratuits / OSS :**

  - **Microsoft 365** : versioning + corbeille = inclus
  - **PowerShell / Graph API** : pour bulk restoration
  - **OneDrive Files Restore** : restauration de tout le
    OneDrive a une date donnee

  Pour les PME plus exposees : **Veeam Backup for Microsoft
  365** (10-20 EUR/user/an) = sauvegarde tierce.

  **Le piege "ce sera dans la corbeille" :**

  La corbeille a **une durée limitee** (30-90 jours selon
  plate-forme). Si tu te rends compte 6 mois plus tard, c'est
  trop tard pour la corbeille.

  Solution : sauvegarde tiers avec retention plus longue
  (1-7 ans).

  **Le piege "je vais recover moi-même" :**

  Tentations classiques :
  - **Reinstaller** Windows / changer de PC : detruit les
    données recuperables
  - **Outils de recovery local** : malware potentiel + ne
    fonctionne pas pour cloud
  - **Demander a un collègue** : il peut aussi se tromper

  Solution : appel IMMEDIAT au DSI/IT. Ils ont les outils + 
  l'expérience.

  **La règle d'or :**

  > **Erreur humaine = 40-60% des pertes de données. Action
  > immediate (5 min) + transparence + appel DSI = restauration
  > probable (95% si < 30 jours). Cachement = catastrophe.
  > Droit a l'erreur dans la politique = signalement rapide.
  > Plusieurs niveaux de defense (corbeille, versions,
  > sauvegarde tiers).**
