debrief: |
  Une fuite de credentials est **probable, pas improbable** : il y a des
  centaines de fuites par an publiques + des milliers privees. **HaveIBeenPwned
  recense plus de 13 milliards de comptes leakes** en 2026.

  La regle d'or : **ce n'est pas grave qu'un mot de passe soit dans une fuite,
  ce qui est grave c'est qu'il soit ENCORE actif quelque part**.

  Procedure complete (30 min) :

  1. **Verifier l'ampleur**
     - HaveIBeenPwned (https://haveibeenpwned.com) : rechercher tous tes emails
     - Bitwarden / 1Password / Dashlane : la plupart ont un dashboard de mots
       de passe leakes integre
     - Pwned Passwords : si tu hesites sur un mot de passe specifique, le
       tester sans le transmettre (k-anonymity)

  2. **Changer les mots de passe leakes** sur les sites concernes
     ET tous les sites ou tu as REUTILISE le meme mot de passe
     (confession honnete : il y en a souvent plus que tu pensais)

  3. **Activer la double authentification (MFA)** sur les comptes critiques :
     - Mail principal
     - Banque
     - Reseaux sociaux
     - Comptes professionnels (CRM, ERP, gestion paie)
     - Comptes cloud (Google, Apple, Microsoft)

  4. **Si la fuite touche un mot de passe pro** : signaler a la DSI/RSSI
     pour qu'ils auditent les acces (logs, sessions actives) et notifient
     CNIL si donnees personnelles touchees.

  5. **Activer les alertes futures** : Bitwarden, HaveIBeenPwned, Firefox
     Monitor, Google Password Manager - tous peuvent te prevenir si
     un nouveau leak apparait.

  La regle universelle : **une fuite de credentials n'est pas une honte, c'est
  une donnee**. Ce qui compte c'est ta reaction, pas l'incident lui-meme.
