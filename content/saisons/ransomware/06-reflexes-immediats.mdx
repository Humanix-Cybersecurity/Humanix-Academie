debrief: |
  Les **60 premieres minutes** d'un ransomware sont le moment ou tu peux encore sauver quelque chose. Apres, le chiffrement progresse, les preuves se perdent, les decisions deviennent plus dures.

  **Voici les 6 reflexes des 60 premieres minutes - a memoriser, afficher dans la salle IT, executer sans hesitation :**

  ---

  **REFLEXE 1 - ISOLER LE RESEAU (minutes 0 a 5)**

  Couper la propagation est la priorite absolue.

  - Couper l'acces internet (firewall en mode panique)
  - Debrancher les partages reseau qui ne sont pas encore touches
  - Desactiver le Wi-Fi sur les postes infectes
  - **NE PAS ETEINDRE** les machines infectees (preserve la memoire vive pour le forensique)

  ---

  **REFLEXE 2 - ACTIVER LA CELLULE DE CRISE (minutes 5 a 15)**

  Les bonnes personnes au bon moment.

  - **DG / DAF** : decision strategique, communication externe
  - **DSI / RSSI** : technique, isolation, restauration
  - **DPO** : si donnees personnelles (notification CNIL)
  - **Avocat / juridique** : implications legales (LOPMI 2023, RGPD)
  - **Assureur cyber** : declenche la couverture, mandate les ressources
  - **Communication** : pour eviter la panique interne et preparer l'externe

  Liste des numeros DOIT etre disponible HORS du SI (papier dans le bureau, contact dans le mobile perso).

  ---

  **REFLEXE 3 - APPELER CYBERMALVEILLANCE.GOUV.FR (minutes 15 a 20)**

  Le 0 805 805 817 (gratuit, 7j/7) est ton point d'entree officiel. Ils :
  - Activent un parcours d'aide
  - T'orientent vers le CSIRT regional
  - Si necessaire, mobilisent l'ANSSI
  - Transmettent un dossier au CERT-FR pour analyse de l'attaque

  Pour les organismes essentiels (NIS2) ou opérateurs d'importance vitale : appel direct **CERT-FR** (cert-fr.ssi.gouv.fr).

  ---

  **REFLEXE 4 - DOCUMENTER T0 ET LES FAITS (minutes 15 a 30, en parallele)**

  Le journal d'incident commence ICI, pas plus tard.

  - **Heure precise de decouverte** (T0)
  - **Premiers symptomes observes** : message de rancon, fichiers chiffres, services down
  - **Postes / serveurs concernes** : liste des hostnames, adresses IP
  - **Actions deja prises** : isolation reseau, etc.
  - **Personnes prevenues** : nom, heure, message echange

  Cette documentation servira a la cellule de crise, a Cybermalveillance, a l'assureur, au juge si plainte. Format : fichier texte sur un poste sain, ou papier - JAMAIS sur un systeme infecte.

  ---

  **REFLEXE 5 - PRESERVER LES PREUVES (minutes 30 a 45)**

  L'analyse forensique a venir depend de ce que tu fais maintenant.

  - **NE PAS** redemarrer les postes infectes
  - **NE PAS** lancer d'antivirus
  - **NE PAS** essayer de "reparer" en bricolant
  - **OUI** : sauvegarder les logs systeme avant qu'ils tournent
  - **OUI** : photographier les ecrans (smartphone) - capture de la demande de rancon
  - **OUI** : noter les processus suspects visibles
  - **OUI** : conserver l'email d'attaque s'il existe (sans l'ouvrir)

  Si tu as un EDR (HarfangLab, CrowdStrike, Microsoft Defender for Endpoint), il a deja les telemetries - ne le desinstalle pas.

  ---

  **REFLEXE 6 - COMMUNIQUER EN INTERNE (minutes 45 a 60)**

  Les salaries vont savoir d'une maniere ou d'une autre. Mieux vaut etre la source.

  - **Message clair, court, factuel** : "Un incident cyber est en cours. Le SI est isole pour controle. Pas de cliquage sur les emails. Pas de partage public. On revient vers vous dans 2h."
  - **Pas de panique** : ne pas donner de details techniques, ne pas accuser, ne pas commenter la rancon
  - **Mode degrade** : indiquer ce qui marche encore (telephone, papier) et qui faire suivre
  - **Confidentialite** : pas de communication externe (clients, fournisseurs, presse) sans validation cellule de crise

  ---

  **APRES LES 60 PREMIERES MINUTES :**

  Tu rentres dans la phase de **gestion de crise** (heures 1 a 24) :
  - Qualification fine de l'attaque (groupe, ransomware, vecteur)
  - Decision restauration vs paiement (cf. module 03)
  - Notification CNIL si donnees personnelles touchees (sous 72h)
  - Communication externe controlee
  - Plan de bascule en mode degrade pour les services critiques

  **L'investissement le plus rentable d'une PME en cybersecurite :** afficher les 6 reflexes dans la salle IT, les exercer une fois par an avec une simulation de panne, et identifier les 6 personnes a appeler avant la crise.

  20 € de plastification + 1 demi-journee d'exercice annuelle = potentiellement 250 000 € economises le jour ou ca tombe.
