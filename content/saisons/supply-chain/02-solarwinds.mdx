debrief: |
  L'attaque SolarWinds (officiellement attribuee au groupe APT29 / Cozy
  Bear, GRU/SVR russe) est l'attaque supply chain logicielle la plus
  importante connue. Elle a redefini ce que le monde cyber considere
  comme possible.

  **Chronologie technique :**

  **Septembre 2019** : les attaquants compromettent l'infrastructure
  de developpement de SolarWinds (le pipeline de build Orion). Pas
  d'acces aux données clients - juste au pipeline.

  **Mars 2020** : ils injectent le malware "SUNBURST" (TEARDROP plus
  tard) dans le code source d'Orion AVANT le build officiel. Le
  binaire produit est signe numeriquement par SolarWinds avec le bon
  certificat. Donc valide aux yeux de tous les antivirus et systèmes
  de vérification.

  **Avril-Octobre 2020** : la version compromise est diffusee comme
  mise a jour officielle a tous les clients SolarWinds qui activent
  l'auto-update. ~18 000 entreprises l'installent.

  **Après l'installation** : SUNBURST attend ~14 jours avant de
  s'activer (pour éviter détection lors d'analyses sandbox initiales).
  Ensuite, il fait un beacon DNS vers `avsvmcloud.com` toutes les
  heures. Si l'attaquant le selectionne comme cible, il deploie une
  charge utile secondaire (TEARDROP) qui ouvre une porte derobee.

  **8 decembre 2020** : FireEye decouvre la compromission lors de
  l'investigation interne d'un breach chez eux.

  **13 decembre 2020** : annonce publique. CERT US (CISA) emet une
  emergency directive demandant a TOUTES les agences federales de
  deconnecter immédiatement les serveurs Orion.

  **Janvier 2021** : SolarWinds publie les patches. Investigation
  forensic complete : ~250 entreprises ont été "selectionnees" pour
  exploitation active (les autres ont juste eu le bot dormant).

  **Pourquoi c'était si grave :**

  1. **Signature numérique valide** : tous les outils de détection
     (antivirus, EDR, code signing vérification) ont laisse passer.

  2. **Persistence privilegiee** : SolarWinds Orion tourne avec des
     droits administrateur sur les serveurs surveilles. Donc le
     malware avait acces admin partout.

  3. **Confiance implicite** : SolarWinds est un editeur reconnu
     depuis 1999, ses clients incluent les agences les plus sensibles.
     Personne ne suspectait une mise a jour officielle.

  4. **Selection silencieuse** : sur 18 000 victimes, seules 250 ont
     été activement exploitees. Le malware restait dormant chez les
     autres - donc très difficile a détecter.

  5. **Exfiltration discrete** : les attaquants utilisaient des
     domaines de commande qui imitaient des services legitimes
     (avsvmcloud.com ressemble a Amazon AWS).

  **Les 5 leçons pour TOUS les SI (pas que les victimes SolarWinds) :**

  **1. La signature numérique n'est pas une garantie absolue**

  Un editeur compromis peut signer du malware. La defense doit être
  en couches : signature + comportement + reputation.

  **2. Les outils d'admin sont des cibles privilegiees**

  Tout outil avec droits admin sur ton parc (monitoring, RMM,
  endpoint management) doit être :
  - Cloisonne (subnet dedie, pas full network)
  - Whitelisted (sortie internet limitee aux domaines necessaires)
  - Logged (toutes les actions tracees)
  - Reviewed (certifications de l'editeur, audit code si possible)

  **3. L'auto-update est une lame a double tranchant**

  Pro : tu patchs vite. Contra : un editeur compromis pousse du
  malware sans ton accord.
  Compromis : auto-update sur les outils peu critiques, manuel après
  validation sur les outils admin / sensibles.

  **4. Network egress monitoring**

  Un serveur de monitoring qui doit parler a `internal-mgmt.acme.fr`
  ne devrait JAMAIS se connecter a `avsvmcloud.com` (domaine
  inconnu). Mettre des règles firewall sortantes par defaut ferme,
  open uniquement vers les destinations connues.

  **5. Defense in depth, no trust**

  Adopter le principe **Zero Trust** : ne jamais faire confiance,
  toujours vérifier. Même aux outils internes signes par un editeur
  reconnu.

  **Suite (le précédent qui a tout change) :**

  Après SolarWinds, les États-Unis ont impose :
  - **Executive Order 14028** (mai 2021) : tous les editeurs vendant
    au gouvernement US doivent fournir un SBOM (Software Bill of
    Materials).
  - **CISA Binding Operational Directive 22-01** : audit obligatoire
    de tous les systèmes admin avec acces privilegie.
  - **Programme SCRM-NIST 800-161** : framework de gestion supply
    chain renforce.

  Côté UE : **NIS2** (article 21) reprend ces principes avec
  obligation de gerer la supply chain pour entites essentielles et
  importantes.

  **Le bilan financier :**

  - SolarWinds (la société) : ~75M$ de pertes directes, action en
    bourse divise par 3.
  - SolarWinds CISO (Tim Brown) : poursuivi en 2023 par la SEC pour
    "negligence dans la divulgation des risques sécurité" (case en
    cours).
  - Microsoft : ~30K heures-homme d'investigation, plusieurs comptes
    Azure compromis, code source de produits accede.
  - Total estime des coûts mondiaux : >100B$ selon Brookings (mais
    estimation contestee).
