debrief: |
  La **facture piegee** est l'un des vecteurs preferes des
  attaquants pour cibler la comptabilite. Plus que 60% des
  fraudes au virement (FOVI) commencent par une facture
  malveillante.

  **Les 5 indices d'une facture piegee :**

  **1. Domaine emetteur usurpe ou similaire**

  - **Typosquatting** : `fourn1sseur.fr` (1 au lieu de i),
    `fournisseure.fr` (e final), `fournisseur-fr.com`
  - **Sous-domaine pirate** : `fournisseur.acme-paiement.fr` (le
    vrai serait `acme.fr`)
  - **TLD different** : `.fr` au lieu de `.com`, ou inverse
  - **Compte compromis** : meme le bon domaine peut etre detourne
    si le fournisseur a ete piratte

  **Verification** : avant d'ouvrir, COPIE l'email expediteur,
  COMPARE avec les emails precedents du fournisseur, en cas de
  doute -> telephone au fournisseur via numero connu.

  **2. Sentiment d'urgence injustifie**

  Phrases types :
  - "Urgent : a regler avant le [date proche]"
  - "Penalites de retard si non-paiement sous 48h"
  - "Le service comptabilite menace de couper la livraison"

  L'urgence est le **levier psychologique** de manipulation. Une
  vraie facture ne demande pas une action immediate.

  **3. Demande d'activation de macros (Word/Excel) ou de
  scripts**

  - Un **PDF** n'a JAMAIS besoin de macros (c'est specifique a
    Office)
  - Un fichier `.docx` ou `.xlsx` legitimes ne demande pas
    d'activer les macros pour s'afficher
  - "Cliquez ici pour debloquer le contenu" = piege

  **Action** : ne JAMAIS activer les macros sur un fichier
  exterieur. Configuration recommandee : macros desactivees par
  defaut (politique GPO Microsoft 365).

  **4. Pieces jointes a extension suspecte**

  - **Faux PDF** : `Facture.pdf.exe` (Windows masque l'extension
    `.exe` par defaut)
  - **ZIP avec executable** : `Facture.zip` contenant
    `facture.exe`
  - **Liens fictifs** : "Cliquer ici pour telecharger la facture"
    pointe vers un site pirate

  **Configuration utile** : afficher les extensions de fichier
  par defaut (Windows : Options des dossiers > Affichage >
  Decocher "Masquer les extensions").

  **5. Changement d'IBAN sans annonce**

  - Une facture qui mentionne un nouvel IBAN sans email/telephone
    prealable = TRES suspect
  - Procedure : tout changement d'IBAN doit etre confirme **par
    telephone via numero connu** avant tout virement (cf ep 02 et
    saison fraude-president)

  **Que peut contenir une facture piegee ?**

  **a) Macros malveillantes**

  Le fichier Word/Excel ouvre du code VBA qui :
  - Telecharge un malware (Emotet, Qakbot, IcedID)
  - Installe un keylogger / RAT (Remote Access Trojan)
  - Chiffre les fichiers (ransomware)

  Cas reel : campagne Emotet 2018-2021 = milliards de pertes
  mondiales, propagee massivement par factures piegees.

  **b) Liens vers sites pirates**

  Le PDF contient un lien (clickable) qui mene vers :
  - Faux site de paiement (le client paie l'attaquant, pas le
    fournisseur)
  - Faux site de banque (capture des credentials)
  - Telechargement de malware

  **c) IBAN frauduleux**

  Le PDF est legitime visuellement mais l'IBAN dans le pied de
  page a ete remplace par celui de l'attaquant. Le comptable
  paie sur le mauvais compte.

  **d) Exploit dans le PDF lui-meme**

  Les anciens lecteurs PDF (Adobe Reader pas a jour) avaient des
  failles permettant l'execution de code par simple ouverture.
  Mise a jour Adobe Reader = mesure de base.

  **Procedure de verification avant paiement (5 etapes) :**

  **1. Verification emetteur (5 secondes)**
  - Domaine = exact match avec les factures precedentes ?
  - Email connu dans le carnet d'adresses ?
  - Si doute : grep dans les anciens mails pour confirmer

  **2. Verification du contenu (10 secondes)**
  - Numero de facture suit la sequence habituelle ?
  - Montant coherent avec la commande passee ?
  - Date de facturation logique ?

  **3. Verification IBAN (10 secondes)**
  - IBAN identique aux factures precedentes ?
  - Si nouveau : appel telephonique au fournisseur (numero
    connu, pas celui du mail) pour confirmer

  **4. Verification PJ (10 secondes)**
  - Extension du fichier (vraie .pdf, pas .pdf.exe)
  - Pas de demande d'activer macros / scripts
  - Visualisation en preview (sans ouverture complete) si
    possible

  **5. Validation hierarchique pour montants > seuil**
  - Politique d'entreprise : tout virement > 5000 EUR (ou seuil
    defini) double validation
  - Validation par DAF / dirigeant pour > 50000 EUR

  **Le piege "le PDF a l'air normal" :**

  Un PDF peut sembler 100% authentique visuellement (logo,
  format, mise en page) mais contenir un IBAN modifie ou un
  lien pirate. La verification ne se fait pas a l'oeil, mais sur
  les MEDONNEES (emetteur, IBAN, telephone).

  **Le piege "c'est urgent" :**

  L'urgence est SUSPECTE. Un fournisseur legitime sait qu'une
  PME a des delais de paiement (30j-60j). Les vraies factures
  ne menacent pas, ne pressent pas. Si "urgent" = doublement
  prudent.

  **Cas reel : campagne FOVI sur Pathe (2018) :**

  Pathe Hollande a perdu 19 M EUR via une fraude au virement
  initiee par une serie de factures piegees + emails de "DG".
  Le directeur financier a paye 7 fois en 1 mois sans verifier
  hors-bande. Voir saison fraude-president ep 06 pour le detail.

  **Outils utiles :**

  - **Visionneuse en ligne** : Google Drive ou Microsoft Office
    Online ouvrent le PDF sans risque local
  - **VirusTotal.com** : scan du PDF / executable contre 70+
    antivirus (gratuit)
  - **Sandbox cuckoo / hybrid-analysis** : execution en VM
    isolee pour cas avances
  - **Politique GPO macros** : desactivation par defaut sur tous
    les postes (Microsoft 365 admin center)
  - **Outlook signal phishing** : bouton "Signaler comme
    phishing" qui envoie au RSSI/SOC

  **La regle d'or :**

  > **5 secondes de verification = 50000 EUR economises. Une
  > facture qui presse, qui demande des macros, qui change
  > l'IBAN sans annonce, est un piege. Telephone au
  > fournisseur via numero connu = blindage.**
