debrief: |
  L'**ERP / logiciel comptable** (Sage, Cegid, Pennylane,
  Quickbooks, SAP, Oracle) est l'un des actifs les plus critiques
  de l'entreprise. Il contient :

  - Toutes les donnees financieres
  - Tous les RIB fournisseurs/clients
  - Les donnees de paie (donnees sensibles RGPD)
  - L'historique de toutes les transactions
  - Les acces au logiciel = acces a la tresorerie

  Compromis = perte de controle financier total.

  **Les 8 mesures de securisation essentielles :**

  **1. Comptes nominatifs (1 personne = 1 compte)**

  - **NON** : compte 'admin' partage entre 5 personnes
  - **OUI** : 1 compte par personne (`alice.dupont`, `bob.martin`)
  - **Avantage** : tracability individuelle (qui a fait quoi ?)
  - **Avantage** : revocation immediate si depart d'un employe
  - **Avantage** : pas de risque "le mot de passe a fuite"
    (chacun gere le sien)

  **Implementation** :
  - Creer 1 compte par utilisateur
  - **Supprimer** ou **renommer** le compte partage admin
  - Migration des donnees / ecritures historiques sur les
    nouveaux comptes (la plupart des ERP le permettent)

  **2. MFA obligatoire**

  - TOTP (Google Authenticator, Authy, Microsoft Authenticator)
  - WebAuthn (YubiKey, passkey) pour acces sensibles
  - **Obligatoire** pour : tous les comptes admin, tous les
    comptes ayant acces aux virements

  **Sage / Cegid / SAP** : MFA natif disponible (Sage Cloud,
  Cegid Cloud, SAP S/4HANA Cloud). Pour les versions on-premise
  : module additionnel ou via un IdP (Microsoft Entra, Okta).

  **3. Mots de passe forts + rotation**

  - **Politique** : 14 caracteres minimum, complexes
  - **Pas de partage** entre employes
  - **Gestionnaire de mots de passe** (Bitwarden, 1Password,
    Dashlane)
  - **Rotation** : tous les 6-12 mois pour comptes admin
  - **Pas de mot de passe dans email/Teams** (canal pas chiffre)

  **4. Audit log + revue mensuelle**

  L'ERP enregistre toutes les actions :
  - Connexion / deconnexion (qui, quand, depuis ou)
  - Saisie d'ecriture
  - Modification d'IBAN fournisseur
  - Validation virement
  - Export de donnees
  - Acces aux donnees sensibles (paie)

  **Conservation** : 5 ans (obligation comptable + RGPD article
  30).

  **Revue mensuelle** par DAF + RSSI :
  - Connexions hors heures (3h du matin = suspect)
  - Modifications d'IBAN (cf ep 02)
  - Exports massifs (peut etre exfiltration)
  - Tentatives de connexion echouees (brute force ?)

  **5. Sauvegardes 3-2-1**

  - **3** copies (BDD principale + 2 sauvegardes)
  - **2** supports differents (disque local + cloud / NAS)
  - **1** off-site (au moins 1 sauvegarde hors du site
    physique = anti-ransomware)
  - **Tests reguliers** de restauration (mensuels)

  **Pour Sage / Cegid / Pennylane** :
  - Sauvegarde automatique quotidienne (par defaut chez les SaaS)
  - Off-site : verifier que c'est bien sur un autre datacenter
  - Conservation : 30-90 jours minimum

  **6. Restriction d'acces**

  - **Acces depuis postes professionnels uniquement** (pas
    perso BYOD pour l'ERP)
  - **Acces depuis IPs autorisees** (whitelist : siege + VPN)
  - **Pas d'acces depuis l'etranger** sauf justifie + MFA
    renforce
  - **Pas de partage d'ecran** sur Teams/Zoom avec l'ERP visible
  - **Bloquer copy/paste** des donnees sensibles si possible
    (DLP)

  **7. Mise a jour reguliere**

  - **Patch immediat** des correctifs de securite (CVE)
  - **Version supportee** (Sage 100c < 5 ans, Cegid... )
  - **Notifications** des fournisseurs ERP : abonner le RSSI
    aux newsletters securite

  **Risque sur ERP non maintenu** : exemple SAP HANA 2019, faille
  permettant l'execution de code distant. Patch dispo, mais 30%
  des installations toujours vulnerables 1 an apres.

  **8. RGPD : registre de traitement**

  L'ERP traite des donnees personnelles (employes, candidats,
  clients, fournisseurs).

  Obligations RGPD :
  - **Registre de traitement** (article 30) : declarer le
    traitement comptabilite/paie
  - **Duree de conservation** : 10 ans (comptabilite), 5 ans
    (paie)
  - **Droits des personnes** : acces, rectification, effacement
    (article 15-22)
  - **Notification breach** : sous 72h CNIL si fuite de donnees

  **Le piege "Sage Cloud est securise par defaut" :**

  Vrai et faux. Sage / Cegid / Pennylane SaaS gerent **leur**
  partie (datacenter, sauvegardes, patches OS). Mais TU es
  responsable de :

  - **Configuration des comptes** (nominatifs, MFA, droits)
  - **Politique de mots de passe**
  - **Revue audit log**
  - **Politique RGPD** (registre, duree, droits des personnes)
  - **Politique d'acces** (qui, depuis ou)

  Modele de **responsabilite partagee** (Shared Responsibility
  Model). Le SaaS te donne les outils, toi tu les configures.

  **Le piege "le compte admin partage est plus pratique" :**

  C'est le piege le plus frequent. Avantages percus :
  - Plus simple a memoriser
  - Tout le monde a les memes droits
  - Pas de gestion de comptes

  Inconvenients reels :
  - Aucune tracability (qui a fait quoi ?)
  - Si une personne fuit le mot de passe, tous compromis
  - Si une personne quitte, le mot de passe doit etre change
    pour tous (rarement fait)
  - **Non-conforme** RGPD article 32 (mesures techniques) +
    NIS2 article 21 (mesure 9)

  Solution : 1 compte / personne, gestionnaire de mots de
  passe pour la simplicite.

  **Le piege "on a un firewall, ca suffit" :**

  Le firewall protege le perimetre, pas l'ERP en interne.
  Risques NON couverts :
  - Compte compromis (phishing) : l'attaquant entre via VPN /
    web
  - Insider (employe rancunier) : acces deja legitime
  - Ransomware via email : passe le firewall
  - SQL injection si appli web mal codee : passe le firewall

  Le firewall est UNE mesure parmi 8. Pas suffisant.

  **Cas reels :**

  **Cas 1 - PME Bordeaux 2022 (vol donnees fournisseurs)**

  Compte admin Sage partage entre 6 personnes. Un comptable a
  ete piratte par phishing (mot de passe identique partout). Le
  pirate a accedes a 12 mois de donnees fournisseurs (RIB
  inclus) qui ont ete revendues. Les fournisseurs ont ete
  victimes de FOVI les mois suivants. La PME a paye 50K EUR de
  remboursements + procedures.

  **Cas 2 - Hopital prive Lyon 2023 (ransomware via ERP)**

  ERP Cegid pas patche depuis 3 ans (CVE 2021 exploitee). Le
  pirate a chiffre l'ERP + sauvegardes (sur meme serveur). 18h
  d'indisponibilite. Cout : 800K EUR (recovery + perte
  d'activite + amende RGPD pour fuite de donnees patients).

  **Cas 3 - Editeur SaaS Paris 2024 (insider)**

  Un comptable rancunier (en periode de preavis) a exporte la
  base entiere des fournisseurs (5000 RIBs) avant de partir.
  Aucun audit log consulte. Decouverte 6 mois plus tard quand
  les fournisseurs ont commence a recevoir du phishing cible.

  **Plan d'action (priorise, 30 jours) :**

  | Semaine | Action | Cout |
  |---|---|---|
  | S1 | Inventaire des comptes existants + identification compte partage | 0 |
  | S1 | Creation comptes nominatifs (1/personne) | 0 |
  | S2 | Migration des utilisateurs vers comptes nominatifs | 0 |
  | S2 | Activation MFA (TOTP) sur tous comptes | 0 |
  | S3 | Configuration audit log + revue mensuelle | 0 |
  | S3 | Sauvegarde off-site (cloud) | 50-200 EUR/mois |
  | S4 | Politique d'acces (postes pro uniquement) + IP whitelist | 0 |
  | S4 | Documentation + formation utilisateurs | 0 |

  **Cout total : 50-200 EUR/mois. ROI tres eleve.**

  **Outils gratuits / OSS pour s'aider :**

  - **Bitwarden** (gratuit) : gestionnaire de mots de passe pour
    PME
  - **Microsoft Authenticator / Google Authenticator** : MFA
    TOTP gratuit
  - **YubiKey 5** (45 EUR) : MFA WebAuthn pour comptes admin
  - **Wazuh** (OSS) : SIEM gratuit pour centraliser audit logs
  - **Veeam Backup Free** ou **Restic** : sauvegarde
    automatique off-site

  **Indicateurs de pilotage (KPI) :**

  Mensuel :
  - **Pourcentage de comptes nominatifs** : doit etre 100%
  - **Pourcentage de comptes avec MFA** : doit etre 100%
  - **Comptes inactifs > 90 jours** : doit etre 0 (revoque)
  - **Tentatives de connexion echouees** : moins de 5 par
    compte/mois (sinon brute force)
  - **Modifications IBAN tracees** : 100% revues par DAF

  Annuel :
  - **Test de restauration sauvegarde** : reussi
  - **Audit acces** : conformite a la politique
  - **Mise a jour ERP** : version supportee

  **La regle d'or :**

  > **L'ERP est l'actif financier le plus critique. Comptes
  > nominatifs + MFA + audit log + sauvegardes off-site sont
  > NON-NEGOCIABLES. Le 'compte admin partage' est la fraude
  > qui n'a pas encore eu lieu. Plan d'action 30j = 50-200
  > EUR/mois.**
