debrief: |
  Le **deces numerique** est un sujet trop souvent neglige.
  Les statistiques (Cybermalveillance / CNIL 2024) :

  - **70% des familles** font face a un drame numerique apres
    deces
  - **Donnees perdues** : photos, documents, archives
  - **Comptes orphelins** : non fermes pendant des annees
    (risque vol identite)
  - **Cryptomonnaies** : ~20% des Bitcoin perdus a jamais
    (cles decedees)
  - **Conflits familiaux** : qui peut acceder a quoi ?

  **Pourquoi preparer maintenant :**

  - **Deces inattendus** : accidents, maladie subite
  - **Soulagement** des proches dans une periode difficile
  - **Conservation** du patrimoine numerique (financier,
    sentimental)
  - **Gestion** des comptes (fermeture, transmission)
  - **Volontes** respectees (que faire de chaque chose)

  **Les 6 categories d'actifs numeriques :**

  **Categorie 1 : Comptes financiers**

  - **Banque en ligne** : LCL, BNP, Boursorama
  - **Investissements** : Yomoni, Linxea, Boursorama Vie
  - **Cryptomonnaies** : Binance, Coinbase, Ledger (hardware
    wallet)
  - **Crowdfunding** : Wiseed, October
  - **Bourse** : Degiro, eToro

  **Risque** : sans acces, perte totale (cryptos surtout).

  **Categorie 2 : Comptes courriers**

  - **Gmail / Outlook / Apple Mail**
  - **Boites secondaires**
  - **Facteur** : reception courriers administratifs
    importants

  **Risque** : impossible de gerer la succession sans acces
  email (banques, administrations).

  **Categorie 3 : Stockage / fichiers**

  - **Photos** : iCloud, Google Photos, Amazon Photos
  - **Documents** : Dropbox, OneDrive, Drive
  - **Cles SSH / GitHub** : pour devs

  **Risque** : photos famille perdues, documents importants
  inaccessibles.

  **Categorie 4 : Reseaux sociaux**

  - **Facebook, Instagram, LinkedIn**
  - **Twitter / X**
  - **Snapchat, TikTok**

  **Risque** : comptes orphelins exploites par usurpateurs
  (vol identite posthume), comptes "in memoriam" non geres.

  **Categorie 5 : Abonnements**

  - **Streaming** : Netflix, Spotify, Apple Music
  - **Cloud / SaaS** : iCloud, Microsoft 365, Adobe
  - **Hosting** : OVH, AWS perso (pour devs)
  - **Services** : Amazon Prime, services divers

  **Risque** : prelevements automatiques continuent (a
  identifier et arreter).

  **Categorie 6 : Pro (si independant / dirigeant)**

  - **Comptes pro** : LinkedIn, Twitter pro
  - **Outils metier** : Slack, Trello, Notion
  - **Donnees clients** : si independant
  - **Acces SI** : si dirigeant PME

  **Risque** : continuite d'activite + obligations RGPD/
  contractuelles.

  **Les outils de transmission :**

  **Outil 1 : Gestionnaire de mots de passe avec acces
  d'urgence**

  Recommande **fortement** :

  **1Password Family / Business** :
  - **Emergency Kit** : fichier PDF avec cle de recuperation
  - Donne au notaire / personne de confiance
  - Permet acces post-mortem au coffre

  **Bitwarden** :
  - **Emergency access** : designation d'une personne
  - Apres delai (7-30 jours sans connexion), acces accorde
  - Gratuit en version Premium personnelle

  **Dashlane** :
  - **Plan d'urgence** : designation contact
  - Procedure post-mortem documentee

  **Cout** : 30-100 EUR/an pour les versions familiales.

  **Outil 2 : Designation "Heritier numerique" sur les
  plateformes**

  Plusieurs plateformes proposent designation officielle :

  **Apple - Legacy Contact (iOS 15.2+)** :
  - Reglages > Apple ID > Legacy Contact
  - Designer 1+ contact qui pourra acceder a iCloud apres
    deces
  - **Apres deces** : Apple verifie + donne acces

  **Google - Inactive Account Manager** :
  - myaccount.google.com > Inactive Account Manager
  - Apres X mois sans connexion : alerte aux contacts
    designes
  - Ils peuvent telecharger les donnees ou supprimer le compte

  **Facebook - Contact Heritier** :
  - Reglages > Contact heritier
  - Apres "memorialisation" : peut gerer (pas tout, mais
    fermer / commemorer)

  **Microsoft 365 - Contacts d'urgence** : permet acces
  apres procedures.

  **Outil 3 : Testament notarial**

  Pour les enjeux importants (cryptos, gros patrimoine) :

  - **Testament olographe** (manuscrit) ou **notarial**
  - **Mention specifique** des actifs numeriques
  - **Cles** stockees chez le notaire (sceau scelle)
  - **Coffre-fort numerique notarial** : nouveaute (plusieurs
    notaires proposent)

  Cout : 100-500 EUR pour un testament basique.

  **Outil 4 : Coffre-fort numerique succession**

  Solutions specialisees :

  - **Testament.fr** : 49-99 EUR/an
  - **Eternime** (US) : pour heritage digital
  - **AfterVault** : conservation post-mortem
  - **Notaire** : services emergents

  **Outil 5 : Instructions ecrites pour les proches**

  Document simple (5-10 pages) qui inclut :

  - **Liste** des comptes (sans mots de passe directement)
  - **Procedure** pour acceder (gestionnaire MDP +
    emergency access)
  - **Volontes** : que faire de chaque compte (fermer,
    memoriser, transmettre)
  - **Contacts** importants (notaire, comptable, banque)
  - **Localisation** des documents physiques (testament,
    cle de coffre)

  **Lieu** : connu par 1-2 proches de confiance + copie au
  notaire.

  **Procedure complete recommandee :**

  **Etape 1 : Inventaire (1 weekend)**

  Liste **TOUS** tes comptes :
  - Email pro / perso
  - Banques (compte courant, livret, assurance vie)
  - Cryptos (Binance, Coinbase, hardware wallets)
  - Stockage (iCloud, Drive, Dropbox)
  - Reseaux sociaux
  - Abonnements (Netflix, Spotify, Amazon Prime, etc.)
  - Services pro (LinkedIn, GitHub, Slack)
  - Sites e-commerce (Amazon, etc.)

  **Outils** : tableur Excel, document Notion, papier.

  **Etape 2 : Categorisation**

  Pour chaque compte :
  - **Categorie** (banque, social, etc.)
  - **Importance** : critique / important / secondaire
  - **Volonte** : transmettre / fermer / memoriser

  **Etape 3 : Securisation**

  - **Gestionnaire de mots de passe** unique pour tous
    (1Password, Bitwarden, Dashlane)
  - **MFA** active sur les critiques
  - **Mots de passe forts** + uniques

  **Etape 4 : Designation des heritiers numeriques**

  - **Apple Legacy Contact** : configurer
  - **Google Inactive Account Manager** : configurer
  - **Facebook Contact heritier** : configurer
  - **1Password / Bitwarden** : Emergency access

  **Etape 5 : Documentation pour les proches**

  Document de 5-10 pages :
  - Liste comptes (sans mots de passe directement)
  - Procedure d'acces
  - Volontes par compte
  - Contacts importants

  **Etape 6 : Testament notarial (si besoin)**

  Pour patrimoine numerique important :
  - Mention specifique des cryptomonnaies
  - Cles ou seed phrases
  - Volontes precises

  **Etape 7 : Communication aux proches**

  - **2-3 personnes de confiance** informees
  - **Localisation** des documents
  - **Procedures** d'acces

  **Etape 8 : Mise a jour annuelle**

  - **Revue annuelle** : nouveaux comptes, departs
  - **Tests** : verifier que les emergency contacts marchent
  - **Mise a jour** des volontes

  **Procedure post-deces (pour les proches) :**

  Si un proche decede et il n'avait rien prepare :

  **Heure 0 - Jour 7 :**
  - Acte de deces (mairie)
  - Notification banque (souvent 24-48h apres acte)
  - Informer assureurs (assurance vie, sante)

  **Jour 7 - 30 :**
  - Inventaire des comptes connus
  - Demande d'acces aux plateformes (procedure souvent
    longue)
  - Notaire : gestion succession

  **Mois 1-3 :**
  - Recuperation des donnees (si possible)
  - Fermeture des comptes
  - Memorisation des reseaux sociaux (Facebook, Instagram)

  **Mois 3-12 :**
  - Suite de la succession (notaire)
  - Gestion patrimoine numerique
  - Cryptomonnaies (souvent perdues si pas de seed)

  **Quelques procedures specifiques :**

  **Apple iCloud post-mortem :**

  Si Legacy Contact pas designe :
  - Demande aupres d'Apple avec acte de deces
  - Procedure judiciaire si necessaire
  - **6-12 mois** typique
  - Souvent : refus si protections renforcees

  **Google :**
  - Procedure "Demande pour un proche decede"
  - Telechargement des donnees possible
  - **Mois** typique

  **Facebook :**
  - **Memorialisation** : compte transforme en memorial
    (visible mais figé)
  - **Suppression** possible aussi
  - Demande proches via formulaire

  **Banque francaise :**
  - **Acte de deces** = blocage du compte
  - **Notaire** gere la succession
  - **Heritiers** recoivent les fonds

  **Cryptomonnaies (cas le plus difficile) :**
  - **Binance / Coinbase** : procedure judiciaire complexe
  - **Hardware wallets (Ledger, Trezor)** : si pas de seed,
    fonds **perdus a jamais**
  - **Solution** : seed phrase chez notaire ou en testament

  **Cas reels :**

  **Cas 1 - Famille 2023 (preparation efficace)**

  Pere de famille 45 ans avait :
  - 1Password Family avec Emergency access pour epouse
  - Apple Legacy Contact
  - Facebook contact heritier
  - Document instructions chez notaire

  Deces accidentel. Apres 1 mois :
  - Epouse a acces a tous les comptes
  - Photos famille recuperees (iCloud)
  - Comptes geres (fermer / memoriser)
  - Cryptos transferees (cles dans testament)

  Pas de drame numerique. Famille soulagee.

  **Cas 2 - Famille 2022 (cryptos perdues)**

  Pere 50 ans avec 100K EUR de Bitcoin sur Ledger Nano. Pas
  de transmission de la seed phrase.

  Deces. Famille a Ledger physiquement mais :
  - Seed phrase introuvable
  - Pas de PIN
  - **Bitcoin perdus a jamais**

  Lecon : seed phrase = chez notaire ou en testament.

  **Cas 3 - PME independant 2024 (continuite affaires)**

  Independant 60 ans (consultant) :
  - 50 clients actifs
  - Donnees clients sur OneDrive / Dropbox
  - GitHub avec code

  Deces. Aucune preparation numerique :
  - Clients perdus (impossible de transferer dossiers)
  - Code GitHub : compromis (pas d'acces)
  - Famille en difficulte (revenus zero)

  Cout : -100K EUR de patrimoine + chaos administratif.

  Solution preventive : delegation officielle a un confrere
  + emergency access aux comptes pro.

  **Cas 4 - Reseaux sociaux post-mortem (cas frequent)**

  Sans designation de heritier :
  - Facebook actif des annees apres deces
  - **Faux comptes** crees au nom du defunt
  - **Souvenirs** de proches mal geres
  - **Spam / hacking** du compte orphelin

  Solution : designation Contact heritier obligatoire pour
  tous + memorisation post-deces.

  **Le piege "ma femme connait tout" :**

  Souvent FAUX :
  - Tu as 50-200 comptes en ligne, ton/ta partenaire en
    connait 5
  - Mots de passe stockes dans ton navigateur (sync iPhone) :
    inaccessibles sans ton iPhone deverouille
  - Cryptos / hardware wallets : seed phrase rarement partagee
  - Procedures variees selon plateformes

  Solution : gestionnaire de mots de passe + emergency access
  + documentation.

  **Le piege "j'ai tout note dans un cahier" :**

  Limites :
  - **Pas de mises a jour** automatiques
  - **Perte / destruction** possible
  - **Acces non-controle** par tiers (entree dans la
    chambre)
  - **Pas de procedure** post-mortem claire

  Mieux : gestionnaire mots de passe + papier en backup chez
  notaire.

  **Le piege "je n'ai pas envie d'y penser" :**

  Comprehensible mais :
  - **Soulage** les proches dans une periode difficile
  - **Preserve** le patrimoine
  - **Respect** des volontes
  - **Eviter** les conflits familiaux

  C'est un acte de **consideration pour les proches**.

  **Cout total recommande pour preparer :**

  - **1Password Family** : 60-90 EUR/an
  - **Testament notarial basique** : 100-300 EUR (one-shot)
  - **Apple/Google Legacy Contact** : gratuit
  - **Document instructions** : gratuit

  Total : ~150 EUR initial + 60 EUR/an. Tres faible vs
  l'impact pour les proches.

  **La regle d'or :**

  > **Le deces numerique est inevitable. Preparation = acte
  > de soin pour les proches. Inventaire + gestionnaire MDP
  > avec emergency access + designation heritiers numeriques
  > (Apple, Google, Facebook) + testament notarial pour gros
  > enjeux + instructions ecrites. Cout : ~150 EUR initial.
  > Soulage drastiquement la famille en cas de deces.**
