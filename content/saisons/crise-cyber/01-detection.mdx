debrief: |
  La détection est la **première ligne de defense après compromission**.
  Plus l'attaque est detectee tot, moins les degats sont élèves.

  Statistique IBM Cost of Data Breach 2024 : un breach detecte sous
  200 jours coûte en moyenne 1.4M EUR ; au-dela = 4.5M EUR. Donc
  détecter en 1 mois plutôt qu'en 1 an peut sauver des millions.

  **Les 3 catégories de signaux faibles :**

  **1. Signaux LOGS (techniques)**

  Anomalies dans les logs système/réseau/application :

  - **Connexions a des heures inhabituelles** : connexion VPN a 3h du
    matin alors que l'employé est en France et le service ouvre a 9h.
  - **Connexions depuis des IPs inhabituelles** : login depuis une IP
    russe pour un user qui n'a jamais voyage.
  - **Pic de trafic sortant** : un serveur qui exfiltre normalement
    100 Mo/jour sort soudainement 50 Go.
  - **Multiples echec d'auth puis succes** : pattern classique de
    brute force / credential stuffing.
  - **Modifications de règles email** (Outlook, Gmail) : "tous les
    mails de banque -> dossier archive" = signature d'un attaquant
    qui veut cacher les notifications.
  - **Nouveaux process / services demarres** : services Windows
    inconnus, processus avec noms ressemblant a des legitimes
    (svchosst.exe au lieu de svchost.exe).

  **2. Signaux UTILISATEUR (humains)**

  Indices que les utilisateurs eux-mêmes peuvent reporter :

  - **Mails 'envoyes' qu'ils n'ont pas envoyes** (cas du scenario).
  - **Lenteur inhabituelle** du poste.
  - **Pop-ups bizarres**, browser qui se redirige vers des sites
    inconnus.
  - **Alertes "votre mot de passe a été change"** d'un service auquel
    l'utilisateur n'a pas touche.
  - **Demandes de paiement urgentes** d'un collègue (fraude au
    président variant, cf. saison fraude-président).

  Ces signaux sont souvent ignores ("c'est lent, c'est normal mon PC
  est vieux") - alors qu'ils peuvent indiquer une compromission.

  **3. Signaux EXTERNES**

  - **Notifications de breach prestataire** (cf. saison supply-chain).
  - **Have I Been Pwned** : leaks de credentials reportes.
  - **Alerte EDR / antivirus** : détection de malware actif.
  - **Plainte client** : "je viens de recevoir un mail suspect de
    votre boîte avec un lien bizarre".
  - **Surveillance dark web** : tes credentials/données leak en
    vente.

  **Distinguer signal du bruit :**

  Tous les jours, des centaines d'alertes mineures se produisent
  dans une organisation. La cle est la **convergence** :
  - Un seul signal = probablement bruit (faux positif, maintenance,
    erreur user)
  - Deux signaux concordants = a investiguer
  - Trois signaux concordants = haute probabilite d'incident reel

  Dans le scenario : mail envoye sans toi + règle de tri non creee
  + lenteur = **3 signaux convergents** = compromission de compte
  hautement probable. Action immediate.

  **Outils de détection (par tier de maturite) :**

  **Tier 1 - Gratuit / facile (PME starter)**
  - **Microsoft 365 Audit Log** : detecte les modifications de règles,
    les connexions inhabituelles. A activer.
  - **Azure AD sign-in logs** : geolocalisation des connexions, MFA
    failures.
  - **Windows Event Log** : événements 4624 (login), 4625 (login
    failed), 4720 (compte cree).
  - **Gmail Activity** : connexions actives, IPs, devices.

  **Tier 2 - Investissement modere (PME mature)**
  - **EDR/XDR** : Microsoft Defender, CrowdStrike, SentinelOne,
    HarfangLab (FR). Detecte les comportements suspects sur les
    postes.
  - **SIEM** : Sekoia.io (FR), Wazuh (gratuit OSS), Elastic Security.
    Centralise et correle les logs.

  **Tier 3 - Mature / grands comptes**
  - **SOC interne ou outsource** (24/7) : surveillance humaine +
    automatisee.
  - **Threat Intelligence** : OpenCTI (FR OSS) pour anticiper les
    menaces.

  **Pour la PME : la règle des 5 minutes**

  Sans EDR/SIEM, les utilisateurs sont tes capteurs. Il faut une
  procédure simple :
  1. Si tu observes un comportement anormal, **5 minutes** pour
     vérifier.
  2. Si confirmation = appel direct au DSI/RSSI (pas mail, pas chat).
  3. Le DSI/RSSI declenche le runbook incident (cf. ep 02 cellule
     de crise).

  Sensibilise les utilisateurs : "en cas de doute, appelle". Mieux
  vaut 10 faux positifs que 1 incident detecte 6 mois trop tard.

  **Cas reel - Equifax 2017 :**

  Equifax a été compromis en mars 2017. Detecte... fin juillet 2017.
  4 mois pour détecter une exfiltration de 147M de dossiers
  personnels. Cause : signaux d'alerte ignores par l'équipe SOC,
  patches Apache Struts non appliques, pas de SIEM correle.

  Cout final : 1.4 milliards $ (amendes + class actions + remediation).
