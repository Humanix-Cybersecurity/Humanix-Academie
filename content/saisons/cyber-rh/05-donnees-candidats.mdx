debrief: |
  La gestion des **données de candidats** est l'un des
  traitements les plus sensibles RGPD. Cumule :

  - **Données personnelles** (identifiantes, contacts)
  - **Données parfois sensibles** (sante, opinions politiques
    si mentionnees)
  - **Volume élève** : centaines/milliers de CV
  - **Risque juridique** : discrimination si pratiques mal
    documentees

  **Le cadre RGPD applicable :**

  **Article 5 RGPD : Principes**

  - **Licielite** : base legale claire (consentement,
    contrat, etc.)
  - **Loyaute, transparence** : information claire des candidats
  - **Limitation des finalites** : recrutement uniquement
  - **Minimisation** : uniquement les données necessaires
  - **Exactitude** : données mises a jour
  - **Limitation de conservation** : pas plus longtemps que
    necessaire
  - **Integrite, confidentialité** : protection technique et
    organisationnelle
  - **Responsabilite** : preuve de conformité (registre)

  **Article 6 : Bases legales**

  Pour les données de candidats, deux bases legales possibles :

  - **Article 6(1)(a) - Consentement** : le candidat accepte
    explicitement
  - **Article 6(1)(b) - Mesures pre-contractuelles** : pour
    les candidats qui ont vraiment postule (pas spontanee)
  - **Article 6(1)(f) - Interet legitime** : très encadre,
    documenter le test 3 étapes

  **Recommandation CNIL** : utiliser le **consentement** (le
  plus protecteur) ou les **mesures pre-contractuelles**.

  **Article 13-14 : Information**

  Le candidat doit être informe **au moment** de la collecte :

  - Identite du responsable de traitement (l'employeur)
  - Coordonnees du DPO (si > 250 employés)
  - Finalites (recrutement)
  - Base legale
  - Duree de conservation
  - Destinataires (RH, manager, comite recrutement)
  - **Droits** (acces, rectification, effacement, opposition)
  - Coordonnees CNIL pour reclamation

  **Format** : "Politique de protection des données" (1-2 pages
  claires) sur le site de candidature ou dans l'ATS.

  **Les durées de conservation recommandees :**

  | Statut | Duree max | Base legale |
  |---|---|---|
  | **Candidat en cours d'entretien** | Jusqu'à fin du process + 30j | Mesures pre-contractuelles |
  | **Candidat non-retenu (avec consentement)** | 2 ans max | Consentement |
  | **Candidat non-retenu (sans consentement)** | Suppression immediate après process | Mesures pre-contractuelles |
  | **Candidat embauche** | Duree des droits + 5 ans | Contrat de travail |
  | **Ex-employé** | Duree des droits + 5 ans | Conformite legale |
  | **Candidature spontanee (avec consentement)** | 2 ans max | Consentement |
  | **Candidature spontanee (sans consentement)** | Suppression immediate | - |

  **Le piege "on garde au cas ou" :**

  Garder des données "au cas ou" = non-conforme. La CNIL
  considere que :

  - Au-dela de **2 ans**, très peu probable de re-utiliser
  - Si besoin futur, le candidat sera plus a jour avec un
    nouveau CV
  - **Risque** : sanctions en cas de plainte ou audit

  Sanctions possibles :
  - Avertissement CNIL
  - Mise en demeure (publique)
  - Sanction financiere (jusqu'à 2% CA)

  **Procédure de gestion conforme :**

  **Étape 1 : Collecte transparente**

  - Politique de protection des données affichee
  - Consentement explicite a la conservation > durée minimale
  - **Case a cocher** explicite : "J'accepte la conservation de
    mon CV pour 2 ans pour vous permettre de me proposer
    d'autres opportunites"

  **Étape 2 : Stockage securise**

  - **ATS** dedie (Applicant Tracking System) : Lever, Greenhouse,
    Workable, Recruitee
  - **Acces restreint** : RH + manager du poste uniquement
  - **MFA** sur l'ATS (données sensibles)
  - **Audit log** des acces et modifications

  **Étape 3 : Information continue**

  - **A chaque étape**, le candidat sait ou il en est
  - **Refus motive** envoye (mais pas obligatoire de detailler)
  - **Information** sur la durée de conservation post-refus

  **Étape 4 : Suppression automatique**

  - **2 ans après non-retention** : suppression automatique
    (calendrier dans l'ATS)
  - **Suppression sur demande** (article 17 RGPD - droit a
    l'oubli) : sous 30 jours
  - **Documentation** de la suppression (preuve)

  **Étape 5 : Registre RGPD (article 30)**

  Pour le traitement "candidatures" :

  - **Finalite** : recrutement
  - **Base legale** : consentement / mesures pre-contractuelles
  - **Personnes concernees** : candidats
  - **Categories de données** : nom, coordonnees, parcours,
    diplomes
  - **Destinataires** : RH, manager du poste, ATS provider
  - **Duree de conservation** : 2 ans non-retention (avec
    consentement)
  - **Mesures de sécurité** : MFA ATS, acces restreint, chiffrement

  **Les 7 droits des candidats RGPD :**

  **1. Droit d'acces (article 15)**
  - "Quelles données avez-vous sur moi ?"
  - Réponse sous 30 jours, gratuit

  **2. Droit de rectification (article 16)**
  - "Cette donnee est fausse, corrigez-la"

  **3. Droit a l'effacement / oubli (article 17)**
  - "Supprimez toutes mes données"
  - **Sauf** si vous etes en cours de process (mesures
    pre-contractuelles)

  **4. Droit a la limitation (article 18)**
  - "Conservez mais n'utilisez pas"

  **5. Droit a la portabilite (article 20)**
  - "Donnez-moi mes données dans un format reutilisable"

  **6. Droit d'opposition (article 21)**
  - "Je m'oppose au traitement"
  - Doit être justifie

  **7. Droit lie aux décisions automatisees (article 22)**
  - Si l'ATS pre-filtre les CV par algorithme : information
    + intervention humaine possible

  **Les pieges du recrutement RGPD :**

  **Piege 1 : "Pre-screening" automatise par IA**

  Beaucoup d'entreprises utilisent des outils IA pour
  pre-filtrer les CV. Risques :

  - **Discrimination algorithmique** (cf cas Amazon : IA
    discriminait les femmes en 2018, abandonnee)
  - **Décision sans intervention humaine** = article 22 RGPD
  - **Opacite** : le candidat ne sait pas pourquoi il est
    refuse

  Solution : intervention humaine obligatoire avant décision
  finale + transparence sur l'usage IA.

  **Piege 2 : Réseaux sociaux du candidat**

  Recherches Google / LinkedIn / Facebook du candidat :

  - **LinkedIn pro** : OK (le candidat l'a publie pour ce but)
  - **Facebook prive** : pas OK (données privées, vie privée)
  - **Twitter** : zone grise, depends du contenu

  Recommandation : limiter aux réseaux pro (LinkedIn).
  Documenter les pratiques.

  **Piege 3 : Tests de personnalite / psychotechniques**

  - **Données sensibles** (article 9 RGPD) : opinions, sante
    mentale
  - **Consentement explicite** obligatoire
  - **Conservation** : strictement limitee (3 mois max)
  - **Acces** : restreint au psychologue / RH responsable

  **Piege 4 : Données sensibles non-demandees mais reçues**

  Le candidat envoie son CV avec :
  - Mention de sa religion
  - Photo (interdite de demande mais pas de reception)
  - Mention de handicap
  - Données de famille

  Action : **NE PAS UTILISER** ces données pour la décision.
  Si trace, supprimer. Documenter pour preuve de
  non-discrimination.

  **Piege 5 : Discrimination cachee**

  - Filtrage par photo (apparence, age, origine)
  - Filtrage par adresse (quartier sensible)
  - Filtrage par nom (origine ethnique)
  - Filtrage par age (date de naissance)

  Tous **interdits**. Sanctions très severes (penal +
  RGPD + Code du travail).

  Solution : **CV anonymises** (sans photo, sans adresse, sans
  age, sans nom) pour le pre-screening.

  **Outils ATS et conformité RGPD :**

  | ATS | Conformite RGPD | Cout PME |
  |---|---|---|
  | Lever | OK (US, mais EU compliant) | 100-200 EUR/mois |
  | Greenhouse | OK | 200-500 EUR/mois |
  | Workable | OK | 80-150 EUR/mois |
  | Recruitee (FR) | OK + DPO support | 100-300 EUR/mois |
  | Talentlyft | OK + suppression auto | 80-150 EUR/mois |
  | Welcome to the Jungle (FR) | OK | 200-500 EUR/mois |

  **Cas reels :**

  **Cas 1 - PME 80 employés 2023 (CNIL warning)**

  Audit CNIL aleatoire sur les pratiques RH. Constat : 1500
  CV conserves depuis 2015 (8 ans) sans base legale. Sanction
  : warning + obligation de suppression sous 3 mois + audit
  annuel.

  Cout : audit CNIL + suppression manuelle (pas d'ATS) = 5000
  EUR de prestations.

  **Cas 2 - ETI 500 employés 2024 (sanction)**

  Plainte d'un candidat refuse 4 ans plus tot dont les données
  étaient toujours conservees. Sanction CNIL : 50 K EUR +
  publication sur le site CNIL.

  **Cas 3 - PME 30 employés 2022 (discrimination)**

  Recruteur supprimait systematiquement les CV avec photos
  de personnes plus de 50 ans. Plainte d'une candidate refusee
  + audit CNIL. Sanctions cumulees :
  - CNIL : 30 K EUR
  - Code du travail : 1 an de prison + 45 K EUR amende
    (responsable)
  - Reputation : presse locale + publication CNIL

  **La règle d'or :**

  > **Données candidats = consentement explicite + durée
  > limitee (2 ans max post-refus) + suppression automatique +
  > registre RGPD + acces restreint. CV anonymises pour le
  > pre-screening = anti-discrimination. ATS dedie = conformité
  > simplifiee (~100-200 EUR/mois pour PME).**
