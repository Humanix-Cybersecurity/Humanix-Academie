debrief: |
  Le **offboarding** d'un employe est l'etape la plus negligee
  du cycle de vie RH/IT. Le contrat est termine, l'attention
  est ailleurs, et les acces techniques restent actifs pendant
  des mois ou des annees.

  Verizon DBIR : **60% des breaches insider** partent d'un
  compte non-revoque correctement.

  **Pourquoi le offboarding est critique :**

  Un compte non-revoque peut :

  1. **Etre compromis par un attaquant** : credentials sur le
    dark web, brute force
  2. **Etre exploite par l'ex-employe rancunier** (cas Tesla
    2018, 300 Go exfiltres 6 mois apres depart)
  3. **Etre utilise par un concurrent** qui aurait recrute
    l'ex-employe
  4. **Servir de back-door** pour ransomware / espionnage

  Sans procedure : risque persistant pendant des annees.

  **La checklist exhaustive (20 points) :**

  **Phase 1 : J-3 a J-1 (Preparation)**

  **1. Notification RH au IT**
  - Email officiel : nom, dernier jour, raison (departure type)
  - Coordination : RH + IT + manager + comptabilite

  **2. Plan de transition**
  - Successeur identifie (ou re-attribution des dossiers)
  - Documentation des projets en cours
  - **Knowledge transfer** : 1-2h avec successeur

  **3. Inventaire materiel**
  - Liste des materiels remis (laptop, telephone, badge,
    cles, USB)
  - Numeros de serie a verifier le J0

  **4. Inventaire acces logiques**
  - Tous les comptes nominatifs (AD, SSO, apps SaaS)
  - Tous les comptes partages dont il connaitrait le mot de
    passe
  - Toutes les apps tierces (CRM, ERP, outils metier)

  **Phase 2 : J0 (Dernier jour) - Action immediate**

  **5. Revocation des comptes nominatifs**

  - **Active Directory / Microsoft Entra** : disable + revocation
    sessions actives
  - **SSO** : disable
  - **Email** : disable + redirection vers le successeur
    (politique 30j)
  - **VPN** : revocation certificat + compte
  - **MFA** : suppression des dispositifs lies (telephone,
    YubiKey)

  **Action rapide** : la revocation doit etre faite **dans
  l'heure** qui suit le depart effectif. Pas a la fin de la
  semaine.

  **6. Revocation des comptes SaaS / apps tierces**

  - Salesforce, HubSpot (CRM)
  - Slack / Teams (avec exclusion des canaux prives)
  - Jira / Confluence / Notion (acces projets)
  - GitHub / GitLab (code source !)
  - **Tous les outils metier** (liste a verifier)

  **7. Mots de passe partages a changer**

  Si l'employe avait acces a des comptes partages (rare mais
  arrive en PME) :
  - Compte support fournisseur
  - Compte reseaux sociaux entreprise
  - Compte Wifi (WPA2 partage)
  - **Tout mot de passe** que l'employe pourrait connaitre

  **8. Recuperation materiel**

  - Laptop (PV de remise)
  - Telephone professionnel
  - Badge d'entree (a desactiver immediatement)
  - Cles physiques bureau
  - Cles USB / disques durs
  - **Verifier que rien n'est manquant**

  **9. Reset du materiel**

  - Wipe complet du laptop (re-image OS)
  - Reset usine du telephone
  - Sauvegarde des donnees pro avant wipe (selon politique)

  **10. Sortie escortee** (selon contexte)

  Pour les departures sensibles (licenciement, conflit) :
  - Sortie escortee par le manager
  - Acces aux locaux desactive avant sortie
  - Pas de retour aux bureaux apres sortie

  **Phase 3 : J+1 a J+7 (Suivi proche)**

  **11. Audit des acces residuels**

  Verifications :
  - Aucune connexion VPN apres le J0
  - Aucune connexion AD / SSO
  - Aucun email entrant/sortant
  - Aucun acces aux apps SaaS

  Si trace de connexion = blocage immediat + investigation.

  **12. Comptes oublies courants**

  Liste des comptes typiquement oublies :
  - Comptes personnels Office 365 / Google qui ont eu acces
    organisation (guest/external)
  - Acces aux backups / shares NAS
  - Comptes API sur services internes (CI/CD, monitoring)
  - Tokens d'authentification (OAuth, JWT)
  - Comptes test / dev / staging
  - Acces aux dashboards (analytics, BI)

  **13. RGPD : donnees personnelles de l'ex-employe**

  - Verifier la duree de conservation des donnees employe
  - Bulletins paie : 50 ans (obligation)
  - Dossier personnel : duree des droits + 5 ans
  - **Anonymisation** des donnees au-dela de la duree

  **Phase 4 : J+30 (Audit final)**

  **14. Audit complet apres 30 jours**

  Verification approfondie :
  - Aucune connexion sur tous les systemes
  - Aucun acces actif (cumul J0 + comptes oublies J+7)
  - Suppression definitive (pas seulement disable)
  - Documentation finale

  **15. Mise a jour de l'inventaire**

  - CMDB : retrait des assets (laptop, etc.)
  - Documentation interne : mise a jour de l'organigramme
  - **Sauvegardes** : retention selon politique

  **Phase 5 : Periodique (Audit comptes fantomes)**

  **16. Audit trimestriel des comptes inactifs**

  - Liste des comptes sans connexion depuis 90 jours
  - Verification : ils correspondent a des employes actifs ?
  - Si ex-employe : revocation immediate
  - **Indicateur** : nombre de comptes fantomes detectes

  **17. Audit annuel exhaustif**

  - Liste de tous les comptes vs liste des employes actifs
  - Pas d'ecart possible
  - **NIS2 article 21 mesure 9** : controle des acces

  **18. Surveillance dark web**

  - Outils : Have I Been Pwned, dehashed
  - Si credentials de l'entreprise apparaissent : alerte
  - Lien probable : compte non-revoque correctement

  **Cas specifiques :**

  **a) Depart non-conflictuel (demission, retraite)**

  Procedure standard ci-dessus. Pas d'urgence particuliere
  mais respect des delais.

  **b) Licenciement / depart conflictuel**

  Procedure renforcee :
  - **Revocation IMMEDIATE** des le moment de l'annonce (pas
    le J+7)
  - **Sortie escortee** des locaux
  - **Audit forensic** des actions des derniers jours/semaines
  - **Surveillance** des donnees telechargees / emails envoyes
  - **Plainte** si exfiltration detectee

  **c) Depart d'un dirigeant / RSSI / DAF**

  - **Toutes** les procedures + **changement** des mots de
    passe partages dont il connaitrait
  - **Notification** aux fournisseurs critiques (changement
    d'interlocuteur)
  - **Audit** des actions sensibles des 6-12 derniers mois

  **d) Depart d'un developpeur**

  - **Revocation Git/GitHub** : urgence (code source !)
  - **Audit** des commits recents
  - **Rotation** des secrets / cles d'API qu'il aurait pu voir
  - **Verification** des branches publiees / tokens persons

  **Cas reel : Tesla 2018**

  Un sous-traitant Tesla, dont le contrat avait pris fin **6
  mois plus tot**, a utilise ses anciens credentials non-revoques
  pour se connecter au reseau interne et exfiltrer **300 Go**
  de donnees sensibles (code, plans, schemas). Decouvert
  plusieurs semaines apres l'exfiltration.

  Cause : oubli de revocation lors de l'offboarding. Le
  sous-traitant avait un acces VPN qui n'a jamais ete coupe.

  Cout estime : > 100 M USD (perte propriete intellectuelle +
  procedures legales).

  **Cas reel : Capital One 2019**

  Un ex-employe d'AWS (qui hebergait Capital One) a utilise sa
  connaissance interne pour exploiter une mauvaise configuration
  apres son depart. **106 millions de comptes** clients exfiltres.

  Cause : l'ex-employe avait conserve sa connaissance des
  systemes mais aussi probablement quelques credentials non
  revoques.

  Cout : 80 M USD d'amende + reputation.

  **Outils utiles pour le offboarding :**

  - **Microsoft Entra Lifecycle Workflows** : automatise
    revocation J0, suppression J+30
  - **Okta Lifecycle Management** : workflows offboarding
  - **JumpCloud** : IAM + offboarding pour PME
  - **BambooHR + Tray.io** : workflow RH-IT integre

  **Cout : 5-15 EUR/utilisateur/mois pour PME.**

  **Indicateurs de pilotage :**

  Mensuel :
  - **Nombre de departs** dans le mois
  - **Temps moyen** entre depart et revocation complete (cible
    : < 1h pour J0)
  - **Nombre de comptes fantomes** detectes a J+30

  Annuel :
  - **Nombre de breaches** lies a comptes non-revoques (cible
    : 0)
  - **Audit comptes inactifs** : conformite

  **La regle d'or :**

  > **Le dernier jour est trop tard pour preparer. Anticipation
  > J-3, revocation IMMEDIATE J0 (dans l'heure), audit J+30.
  > Aucun acces 'au cas ou'. Documentation systematique. La
  > checklist de 20 points est non-negociable.**
