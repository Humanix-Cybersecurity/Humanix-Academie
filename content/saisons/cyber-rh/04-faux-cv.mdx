debrief: |
  Le **faux CV piege** est un vecteur d'attaque cible vers les
  RH. Particulierement efficace car :

  - Les RH **ouvrent des PJ** en permanence (c'est leur metier)
  - Les CV sont souvent **legitimes** (Word, PDF)
  - L'attaquant peut faire passer un faux profil credible
  - La RH n'a pas toujours formation cyber poussee

  Verizon DBIR : **5-8% des breaches** en PME partent d'une
  candidature piegee.

  **Les 3 types d'attaques par CV :**

  **Type 1 : Malware en piece jointe**

  L'attaquant envoie un CV avec une PJ malveillante :
  - **.docm** (Word avec macros) : macros qui telechargent du
    malware
  - **.zip** ou **.rar** contenant un .exe deguise (.exe avec
    icone PDF)
  - **.pdf** avec exploit (rare mais existe)
  - **.lnk** (raccourci Windows) executant code distant

  **Objectif** : compromission du poste RH = acces aux
  candidats + emails internes + souvent aux donnees paie.

  **Type 2 : CV de "candidat" pour infiltration**

  L'attaquant postule veritablement avec un faux profil pour :
  - Etre embauche et infiltrer l'entreprise (espionnage)
  - Recolter des informations lors des entretiens (reseaux,
    securite, processes)
  - Acceder au SI via le badge / acces de stagiaire

  **Cas reel** : 2024, plusieurs entreprises americaines
  embauchent des "developpeurs" qui se reveleront etre des
  agents nord-coreens (rapport US Treasury).

  **Type 3 : Phishing par email de "candidat"**

  - Pas de malware, juste un email de candidature avec un
    "lien vers le CV" qui mene a un site phishing
  - Capture des credentials (Microsoft 365, ATS, etc.)

  **Les 6 indices d'un CV piege :**

  **1. Demande explicite d'activer les macros**

  > "Activez les macros pour voir l'animation"

  Aucun CV legitime n'a besoin de macros. Drapeau rouge
  immediat.

  **2. Format suspect de la PJ**

  - **.docm** (Word avec macros) : suspect par defaut
  - **.zip / .rar** non sollicite : suspect
  - **Faux PDF** : `CV.pdf.exe` (Windows masque .exe par
    defaut)
  - **.lnk / .iso / .img** : exotiques pour une candidature

  **Format normal** : `.pdf` (95% des CV), `.docx` (5%).

  **3. Email expediteur generique / non-coherent**

  - **Adresse generique** : `john123@gmail.com` au lieu de
    `prenom.nom@xxx`
  - **Domaine suspect** : `recruiter-contact.online`
  - **Email recent** (cree pour l'occasion)

  **4. Pas de profil verifiable en ligne**

  - **LinkedIn** : pas de profil, ou profil tres recent (cree
    pour l'occasion)
  - **GitHub** (pour devs) : pas de profil, ou profil vide
  - **Portfolio en ligne** : non fourni ou lien suspect

  **5. Incoherences dans le CV**

  - Photo qui ne correspond pas (recherche image inversee
    Google)
  - **Anciennete impossible** (10 ans d'experience a 22 ans)
  - **Diplomes etranges** ou mal orthographies
  - **Langues : francais maternel** mais email plein de fautes

  **6. Insistance / urgence**

  - **Demande de reponse rapide** ("disponible cette semaine")
  - **Plusieurs relances** rapprochees
  - **Pression** sur le manager pour acceler (suspect pour
    espionnage)

  **Procedure de traitement securisee :**

  **Etape 1 : Pre-tri du mail**

  - Verification de l'expediteur (cf points 3, 4 ci-dessus)
  - Si suspect : pas d'ouverture, signalement RSSI

  **Etape 2 : Visionneuse en ligne**

  Pour les CV qui semblent legitimes mais a verifier :

  - **Office Online** : ouvre les .docx / .pdf sans risque
    local
  - **Google Docs / Drive** : meme principe
  - **Outlook Web** : preview integree
  - **VirusTotal.com** : upload PJ pour scan multi-antivirus
    (gratuit)

  **Avantage** : si le fichier est piege, le malware s'execute
  dans le navigateur sandbox (limite considerablement
  l'impact).

  **Etape 3 : Refus poli des formats suspects**

  Reponse type :

  > "Bonjour, je vous remercie pour votre candidature. Pour des
  > raisons de securite informatique, nous n'ouvrons pas les
  > fichiers .docm ni les archives ZIP. Pourriez-vous nous
  > envoyer votre CV en PDF et votre portfolio en lien
  > LinkedIn / GitHub ? Cordialement"

  Si le candidat refuse / insiste pour le format original :
  drapeau rouge confirme.

  **Etape 4 : Verification du profil**

  - **LinkedIn** : profil + connections + recommendations
  - **GitHub** (devs) : projets + contributions
  - **Recherche Google** : nom + entreprise + ville
  - **Recherche image inversee** : photo coherente avec autres
    profils ?

  **Etape 5 : Verification telephonique avant entretien**

  - Numero du candidat : appel rapide pour confirmer la
    candidature
  - Verification de coherence orale (langue, parcours)
  - Detection de comportement suspect (refus de visio, voix
    qui ne correspond pas)

  **Cas reel : campagne TA505 (2018-present)**

  Le groupe d'attaque TA505 utilise massivement les fausses
  candidatures. Mode operatoire :

  1. Email de candidature en .docm
  2. Macro qui telecharge un dropper
  3. Dropper installe **Get2** (downloader)
  4. **Get2** telecharge **SDBbot** (RAT - acces complet a la
    machine)
  5. **SDBbot** = vol identifiants + reconnaissance + propagation

  Cibles : RH de grandes entreprises et PME.

  **Cas reel : embauche de fausses identites (US 2024)**

  Plusieurs entreprises tech US ont embauche en 2023-2024 des
  "developpeurs" qui etaient en realite des agents
  nord-coreens (rapport Trezasury US). Mode operatoire :

  1. Faux profils LinkedIn avec photos generees IA
  2. CV avec experience inventee (pas verifiable)
  3. Entretiens en visio avec "deepfake live" ou personne
    differente
  4. Une fois embauche : exfiltration code, salaire detourne
    pour financer la Coree du Nord

  **Lecons** : **verifier le profil**, **call video obligatoire**,
  **verification background**, **monitoring** des employes
  recents.

  **Configuration anti-malware pour les RH :**

  **a) Macros desactivees par defaut**

  - **Politique GPO Windows** : macros bloquees pour fichiers
    venant d'internet (Word/Excel)
  - **Microsoft 365 admin center** : protection macro activee
  - **Default deny** : macros bloquees, deblocage seulement
    apres validation IT

  **b) Microsoft Defender for Office 365**

  - **Safe Attachments** : sandbox automatique des PJ avant
    ouverture
  - **Safe Links** : verification des URLs en temps reel
  - **Anti-phishing** : detection des emails frauduleux

  **c) EDR sur le poste RH**

  - **Endpoint Detection Response** (Sentinel One, CrowdStrike,
    Microsoft Defender for Endpoint)
  - **Detection comportementale** : si une macro lance un
    powershell suspect, alerte

  **d) Politique BYOD restrictive**

  - Pas de candidatures sur ordinateur personnel (perte de
    controle anti-malware)
  - **Sandbox dediee** pour ouverture de PJ exotiques

  **Le piege "le candidat insiste, c'est qu'il est motive" :**

  Tres mauvaise lecture. L'insistance, surtout combinee avec
  format suspect, est un drapeau rouge :

  - **Vrai candidat** : envoie un CV PDF, attend la reponse,
    rappelle 1x apres 2 semaines
  - **Faux candidat / attaquant** : insiste, presse, refuse de
    changer de format, multi-relances

  La motivation legitime ne s'exprime pas par la pression sur
  le format de fichier.

  **Le piege "ce sont mes outils metier, je dois ouvrir" :**

  Le metier RH ouvre des PJ tous les jours, mais avec
  **discernement**. La cybersecurite n'empeche pas le metier :

  - **PDF** : ouverture directe (faible risque)
  - **DOCX standard** : visionneuse en ligne (sans risque)
  - **Format suspect** : refus poli + demande PDF

  Pas d'opposition entre "faire son metier" et "ne pas ouvrir
  n'importe quoi".

  **Politique de recrutement cyber-aware :**

  | Etape | Action cyber |
  |---|---|
  | Reception candidature | Pre-tri (email, format, profil) |
  | Lecture CV | Visionneuse en ligne (pas ouverture locale) |
  | Pre-qualification | Verification profil LinkedIn / GitHub |
  | Entretien telephonique | Verification orale (langue, parcours) |
  | Entretien visio | Camera obligatoire + verification identite |
  | Entretien sur site | Badge visiteur (pas badge employe) |
  | Verification references | Appel telephonique au numero officiel (pas mail) |
  | Verification diplomes | Vrai certificat ou attestation |
  | Onboarding | Procedure cyber stricte (cf cyber-rh ep 01) |

  **Indicateurs (KPI) :**

  - **Taux de candidatures suspectes** : 1-3% selon CCI
  - **Taux d'ouvertures de PJ suspectes** : doit etre 0%
  - **Nombre d'incidents** lies a candidature : doit etre 0

  **La regle d'or :**

  > **Une candidature avec demande d'activer les macros, format
  > exotique, ou pression sur le format = phishing potentiel.
  > Visionneuse en ligne pour les CV legitimes. Refus poli des
  > formats suspects + demande PDF. Verification LinkedIn /
  > GitHub avant entretien. Camera obligatoire en visio.**
