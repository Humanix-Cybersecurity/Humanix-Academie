debrief: |
  Le **lien public oublie** est l'une des causes principales de
  fuite de donnees en entreprise. Cumule plusieurs problemes :

  - Facilite d'usage (1 clic pour partager)
  - Pas d'expiration par defaut
  - Pas de notification de fin d'usage
  - Indexation possible par moteurs de recherche

  **Etude UCLA 2022** : 30% des liens publics Google Drive sont
  indexes par Google sous 30 jours.

  **Etude Bitglass 2023** : 7% des entreprises ont au moins 1
  fichier sensible expose publiquement.

  **Comment les liens "secrets" fuitent :**

  **1. Indexation Google / moteurs**

  - Si le document contient certains mots cles, Google peut
    l'indexer
  - Recherche `site:drive.google.com "confidentiel"` = milliers
    de resultats
  - Bots qui crawlent les services cloud

  **2. Forward d'email**

  - Tu envoies le lien a Sophie
  - Sophie forward a Bob (collegue interne ou externe)
  - Bob l'enregistre dans ses bookmarks
  - Bob quitte son entreprise, ses bookmarks sont synchronises
    sur son Mac perso
  - **Le lien circule de plus en plus**

  **3. Capture d'ecran / partage dans messagerie**

  - L'utilisateur partage le lien sur Slack / WhatsApp
  - Capture d'ecran sur Twitter / LinkedIn
  - **Visibilite exponentielle**

  **4. Compromission d'un compte intermediaire**

  - Sophie a recu le lien
  - Compte de Sophie est piratte
  - Le pirate trouve le lien dans ses emails
  - **Acces sans alerte chez toi**

  **5. Scrapers automatises**

  - Bots qui testent en masse des liens type
    `https://drive.google.com/file/d/[hash]/view`
  - Si le hash est court / le pattern previsible : trouvable

  **6. Dark web**

  - Compilation de liens "leaked" vendus en lots
  - Concurrents qui achetent
  - **Acces illimite si pas de revocation**

  **Les 3 niveaux de partage (Google Drive / OneDrive / SharePoint) :**

  **Niveau 1 : Restricted (recommande par defaut)**

  - Acces uniquement aux personnes nommees dans la liste
  - Login requis (Google account, Microsoft account)
  - **Audit log** : qui a accede, quand
  - **Revocation** facile et tracee

  **Niveau 2 : Anyone in the organization**

  - Tous les comptes du tenant Google Workspace / Microsoft 365
    de l'entreprise
  - **Mais** tres rapidement large (250 employes = 250 acces
    potentiels)
  - **Risque** : compte employe compromis = fuite

  Recommandation : utiliser pour les documents non-sensibles
  uniquement (procedures, organigrammes generaux). Pas pour les
  documents confidentiels.

  **Niveau 3 : Anyone with the link (DANGEREUX)**

  - Toute personne avec le lien peut acceder
  - **Pas de login requis**
  - **Pas de tracability** (audit log limite)
  - **Persistant** : tant que le lien existe, accessible

  Recommandation : **JAMAIS** pour les documents sensibles.
  Acceptable uniquement pour :
  - Documents publics (brochures, communiques de presse)
  - Documents one-shot avec **expiration courte** (< 7 jours)

  **Niveau 4 : Public sur le web (catastrophe)**

  - Indexable par les moteurs de recherche
  - Aucune restriction
  - **Eviter absolument**

  **Bonnes pratiques de partage :**

  **1. Toujours nominatif par defaut**

  - Tape l'email du destinataire dans le partage
  - Pas de lien generique
  - Avantage : audit log + revocation facile

  **2. Expiration systematique**

  - Pour les partages externes : expiration **30 jours** par
    defaut
  - Reglage dans Google Workspace / Microsoft 365 admin
  - **Politique organisation** : pas de partage permanent
    externe

  **3. Permissions minimales**

  - **Lecture seule** par defaut (pas commentaire, pas
    modification)
  - Permission a relire que si necessite professionnelle
  - **Pas de "owner"** ou "editor" externe sauf cas particulier

  **4. Notifications**

  - Active les notifications "lien partage"
  - Reception d'un email a chaque acces (pour fichiers
    sensibles)
  - **Audit mensuel** des partages externes

  **5. Audit periodique**

  - **Tous les 3 mois** : revue des partages externes actifs
  - Outils : Google Workspace audit, Microsoft Purview, ou
    scripts admin
  - **Indicateur** : nombre de partages externes / employe

  **Configuration Google Workspace recommandee :**

  Admin Google Workspace > Apps > Drive and Docs > Sharing
  settings :

  - **Sharing outside [organization]** : ON
  - **Whitelist domains** : limiter aux domaines clients/
    partenaires connus (acmecorp.com, etc.)
  - **Default for link sharing** : Restricted (pas Anyone with
    link)
  - **Access checker** : Recipients only
  - **Created in shared drives** : Limited

  **Configuration Microsoft SharePoint / OneDrive :**

  Microsoft 365 admin center > SharePoint > Sharing :

  - **External sharing** : New and existing guests (pas
    Anyone)
  - **Default link type** : Specific people
  - **Default link permission** : View
  - **Expiration date** : 30 jours par defaut

  **Outils d'audit des partages :**

  **a) Google Workspace Investigation Tool**

  - Audit complet des partages externes
  - Filtres : date, domaine, sensibilite
  - Export CSV

  **b) Microsoft Purview**

  - Detection des partages sensibles
  - Politiques DLP
  - Alertes en temps reel

  **c) CASB (Cloud Access Security Broker)**

  - **Microsoft Defender for Cloud Apps** : inclus M365 E5
  - Detection des partages sensibles auto
  - Bloquage des partages externes risques

  **Cas reels :**

  **Cas 1 - PME francaise 2023 (plan strategique fuite)**

  Un commercial a partage le plan strategique 2024 avec un
  prospect via Google Drive "Anyone with link". Le prospect
  l'a transferre a un concurrent. Le concurrent a utilise les
  informations pour soumissions / pricing.

  Decouverte : 6 mois plus tard (perte de contrats inexpliquee).
  Cout estime : 300K EUR de business perdu.

  **Cas 2 - Cabinet d'avocats Paris 2024 (RGPD)**

  Document client (donnees personnelles) partage en "Anyone
  with link" par erreur. Le lien indexe par Google. Decouvert
  par le client (recherche Google de son nom).

  Sanctions :
  - CNIL : 25 K EUR
  - Reputation : -2 etoiles sur Google
  - Perte du client + procedure judiciaire

  **Cas 3 - Hopital prive 2022 (donnees patients)**

  SharePoint avec lien "Anyone in organization" + utilisateur
  externe ajoute par erreur. Donnees patients (1500 dossiers)
  accessibles publiquement pendant 6 mois. Decouvert par
  recherche Google.

  Sanctions :
  - CNIL : 200 K EUR
  - Notification 1500 patients
  - Procedures judiciaires en cours
  - Reputation tres degradee

  **Le piege "le lien est trop long pour etre devine" :**

  FAUX. Les liens Google Drive ressemblent a :
  `https://drive.google.com/file/d/1ABc234XYZ.../view`

  Le hash semble alea mais :
  - **Forwardable** par mail
  - **Capturable** en screenshot
  - **Bots** scrapent en masse
  - **Indexable** si certains mots cles

  Le caractere "long" du lien n'est PAS une protection.

  **Le piege "c'est juste pour 1 personne" :**

  Mais cette personne :
  - Forward a ses collegues
  - Enregistre dans son navigateur (synchro perso)
  - Partage sur Slack / WhatsApp
  - Pourra acceder pour TOUJOURS si pas de revocation

  Solution : partage nominatif + expiration. Si la personne
  veut continuer a acceder, elle doit demander.

  **Le piege "j'ai oublie qui j'ai partage avec" :**

  C'est exactement le risque. Sans audit / outil, impossible
  de retracer 6 mois de partages.

  Solution :
  - Audit periodique trimestriel
  - Outil DLP qui notifie les nouveaux partages
  - Politique : tout partage externe documente dans un tracker

  **La regle d'or :**

  > **Le partage 'Anyone with link' est equivalent a 'public sur
  > internet'. Toujours nominatif + expiration + lecture seule.
  > Audit trimestriel. CASB pour automatisation. Si fuite : 
  > revocation immediate + audit + notification CNIL si donnees
  > personnelles.**
