debrief: |
  L'**USB drop attack** est l'une des attaques les plus simples
  et les plus efficaces. L'attaquant abandonne volontairement
  des cles USB autour d'une entreprise (parking, hall, cafetaria,
  toilettes) en esperant qu'un employé les branche par curiosite.

  **Etudes academiques :**

  - **Blackhat 2016** (etude University of Illinois) : 297 cles
    USB abandonnees, 98% ramassees, 45% branchees, 50% executent
    du code (clic sur un fichier d'etiquette ou similaire).
  - **Google 2019** : 30% des employés branchent une USB trouvee
    selon expériences internes.
  - **CompTIA 2023** : 60% des PME ont subi au moins 1 USB drop
    attack.

  **Pourquoi ça marche si bien :**

  - **Curiosite humaine** : "qu'est-ce qu'il y a dessus ?"
  - **Etiquette tentante** : "salaires", "confidentiel", "photo
    soirée", "concours"
  - **Empathie** : "il faut la rendre au propriétaire"
  - **Convenance** : "j'en avais besoin d'une cle USB"

  **Ce qu'une cle USB malveillante peut faire :**

  **1. Autorun / Autoplay**

  - Sur Windows mal configure : execution automatique d'un .exe
  - Solution : autorun **DESACTIVE par defaut** (politique GPO
    obligatoire)

  **2. BadUSB / Rubber Ducky**

  La cle se fait passer pour un **clavier USB** (HID device)
  et tape automatiquement des commandes a haute vitesse :
  - Ouverture d'un terminal
  - Telechargement d'un malware
  - Execution sans intervention humaine

  L'autorun ne suffit pas a bloquer (l'OS croit que c'est un
  clavier).

  **3. Fichier piege**

  Le fichier "Salaires.docm" sur la cle contient des macros
  qui executent du code (cf saison cyber-rh ep 04).

  **4. USB Killer**

  Cle USB qui envoie une **decharge electrique haute tension**
  qui detruit physiquement le port USB et parfois la carte mère
  (50-200 EUR de degats).

  Cas reels : 2018, etudiant a New York a detruit 66 ordinateurs
  d'une université avec un USB killer (condamnation).

  **5. Implant matériel**

  La cle est en réalité un **point d'acces wifi pirate** ou un
  **dispositif d'écoute audio** dissimule.

  **Le bon reflexe : ne JAMAIS brancher**

  Si tu trouves une cle USB :

  **1. Ne pas la brancher**

  Aucune curiosite ne justifie le risque. Même un petit pourcentage
  de risque sur 10000 actions / an = breach garanti.

  **2. La porter au RSSI / DSI**

  Le service IT a les moyens de :
  - L'examiner sur un poste isole (sandbox / VM)
  - L'analyser pour déterminer s'il y a code malveillant
  - Decider de la suite (poubelle, signalement police si attaque)

  **3. Signaler la decouverte**

  Cle trouvee = signal d'attaque potentielle. Le DSI peut :
  - Vérifier les caméras (qui l'a deposee ?)
  - Alerter les employés (campagne de sensibilisation)
  - Renforcer la surveillance

  **4. Ne pas en parler aux collègues avant le RSSI**

  Pour éviter qu'un autre employé la branche par curiosite (s'il
  l'a vue avant toi).

  **Politique d'entreprise sur l'USB :**

  **Niveau 1 : Sensibilisation**

  - Tous les employés formes : cle inconnue = ne pas brancher
  - Affichage dans les zones de passage

  **Niveau 2 : Politique technique**

  - **GPO Windows** : autorun desactive par defaut
  - **EDR / Antivirus** : détection des USB malveillantes
  - **Politique BYOD USB** : seules les cles de l'entreprise
    autorisees

  **Niveau 3 : Bloquer les ports USB**

  Pour les environnements sensibles :
  - **GPO** : USB en lecture seule sauf pour cles d'entreprise
    chiffrees
  - **Hardware** : ports USB physiquement bloques (caches)
  - **Logiciel DLP** (Data Loss Prévention) : détection
    d'exfiltration

  **Niveau 4 : Tout chiffrer**

  Pour les cles d'entreprise autorisees :
  - **BitLocker To Go** (Windows)
  - **VeraCrypt** (multi-OS, OSS)
  - **Cles matérielles chiffrees** : Kingston IronKey, Apricorn

  **Cas reels :**

  **Cas 1 - Stuxnet (2010, Iran)**

  L'attaque la plus celebre par USB. Une cle USB infectee
  introduit le malware Stuxnet dans les centrifugeuses
  d'enrichissement d'uranium iraniennes (réseau air-gap).
  Résultat : sabotage de centaines de centrifugeuses.

  Lecon : même un réseau air-gap (sans internet) peut être
  compromis par USB.

  **Cas 2 - Hôpital UK 2017**

  Un employé a branche une cle USB trouvee dans le parking. La
  cle contenait un dropper qui a deploye un ransomware sur les
  serveurs. 4 hôpitaux paralyses pendant 48h.

  **Cas 3 - PME française 2022**

  Audit interne : une "cle USB" laissee dans le hall a été
  branchee par 4 employés en 2 jours (avant que le DSI ne la
  recupere). C'était une simulation interne. 4 alertes auraient
  pu être 4 breaches.

  **Variantes a connaître :**

  **a) Cable USB pirate**

  Un cable USB-C ou Lightning peut être **modifie pour pirater**
  (O.MG cable, 100 EUR sur le marche). Apparence : identique
  a un cable Apple/Anker. Fonction : enregistrer les frappes,
  télécharger malware.

  Reflexe : utiliser uniquement TES cables (jamais ceux trouves
  ou pretes).

  **b) Chargeur USB pirate (juice jacking)**

  Borne de chargement publique (gare, aeroport) peut contenir
  un dispositif d'attaque. Cf saison mobile-smartphone ep 06.

  **c) USB factice (clone)**

  L'attaquant fait imprimer des cles USB **avec ton logo
  d'entreprise** et les distribue (salon, conferences).
  L'employé pense que c'est un cadeau officiel.

  Reflexe : se méfier des goodies USB, même avec logo.

  **Pour la maison :**

  Les mêmes règles s'appliquent :
  - Cle USB trouvee dans la rue = poubelle
  - Cle prete par un ami = branche sur un PC dedie / VM
  - Cle d'un magasin (cadeau, photo) = scanner avec antivirus
  - Cle d'occasion (Vinted, Le Bon Coin) = formater avant usage

  **Outils gratuits pour analyser une USB :**

  - **VirusTotal.com** : upload des fichiers de la cle pour
    scan multi-antivirus
  - **MalwareBytes** (gratuit) : scanne specifiquement les
    médias amovibles
  - **VM Linux dedie** : VirtualBox + Ubuntu = environnement
    jetable pour examiner

  Pour les pros : **Cuckoo Sandbox** ou **Hybrid Analysis** :
  analyse approfondie en VM.

  **Le piege "elle est dans un emballage scelle, c'est sur" :**

  FAUX. Les attaquants peuvent :
  - Acheter des emballages d'origine vides (Amazon, eBay)
  - Re-sceller avec film thermo
  - Imprimer leur propre etiquette / packaging

  L'apparence n'est PAS une garantie.

  **La règle d'or :**

  > **Une cle USB inconnue est une menace potentielle. JAMAIS
  > la brancher, même par curiosite. Toujours la porter au
  > DSI / RSSI. Ce reflexe protege l'entreprise et toi-même.
  > L'attaque la plus simple est aussi la plus efficace.**
