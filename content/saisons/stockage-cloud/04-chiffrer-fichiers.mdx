debrief: |
  Le **chiffrement de fichier** est l'une des mesures techniques
  les plus simples et les plus efficaces pour proteger des
  donnees sensibles. Pourtant, 70% des PME ne l'utilisent pas
  systematiquement.

  RGPD article 32 mentionne explicitement le chiffrement comme
  mesure technique de protection.

  **Quand chiffrer ?**

  **Toujours** :
  - Donnees personnelles (CV, salaires, RIB, sante)
  - Donnees commerciales sensibles (plan strategique, prix
    confidentiels)
  - Donnees techniques (code source, secrets, credentials)
  - Documents juridiques (contrats, avenants)
  - Donnees bancaires

  **Selon contexte** :
  - Documents internes envoyes a l'externe
  - Fichiers stockes sur cloud public
  - Fichiers sur cle USB / disque externe

  **Les 3 methodes principales :**

  **Methode 1 : 7-Zip avec mot de passe (gratuit, OSS)**

  **Pour qui** : tous (Windows, Mac via Keka, Linux via p7zip)

  **Procedure** :
  1. Installer 7-Zip (gratuit, www.7-zip.org)
  2. Clic droit sur le fichier > 7-Zip > Add to archive
  3. Format : 7z ou ZIP
  4. **Encryption** : AES-256
  5. **Encrypt file names** : coche (optionnel mais
    recommande)
  6. **Mot de passe** : 14+ caracteres, complexe
  7. OK > archive creee

  **Avantages** :
  - Gratuit, OSS
  - Compatible avec WinRAR, Mac (Keka), Linux
  - AES-256 = chiffrement militaire

  **Inconvenients** :
  - Le destinataire doit avoir 7-Zip / equivalent
  - Le mot de passe doit etre transmis HORS-BANDE
  - Si MDP perdu : fichier inrecuperable

  **Methode 2 : Microsoft Office / LibreOffice avec mot de
  passe**

  **Pour qui** : utilisateurs Office, fichiers Word/Excel/PDF

  **Procedure Word/Excel** :
  1. Fichier > Informations > Proteger le document > Chiffrer
    par mot de passe
  2. Saisir mot de passe (14+ caracteres)
  3. Confirmer
  4. Sauvegarder

  **Procedure PDF** (Adobe Acrobat) :
  1. Outils > Proteger > Chiffrer > Chiffrer par mot de passe
  2. Definir mot de passe d'ouverture + mot de passe de
    permissions

  **Avantages** :
  - Pas de logiciel additionnel
  - Le destinataire doit juste avoir Office / Adobe Reader
  - Compatible avec la plupart des destinataires

  **Inconvenients** :
  - Versions anciennes d'Office (< 2007) avaient un
    chiffrement faible (cassable)
  - PDF avec mot de passe = chiffrement faible (cassable
    rapidement avec outils comme John the Ripper)

  **Methode 3 : Service de chiffrement bout-en-bout**

  **Pour qui** : entreprises avec besoins reguliers

  **Solutions** :

  **a) ProtonMail / Tutanota** (chiffrement email)
  - Gratuit jusqu'a 1 Go (ProtonMail) / 500 Mo (Tutanota)
  - Chiffrement de bout en bout entre comptes
  - Pour destinataire externe : email avec mot de passe
  - **Cout pro** : 5-15 EUR/user/mois

  **b) Tresorit** (cloud chiffre)
  - Drive chiffre de bout en bout
  - Partage avec mot de passe
  - **Conformite GDPR / HIPAA** native
  - **Cout** : 12-24 EUR/user/mois

  **c) Sealit** (FR)
  - Service francais de chiffrement
  - Email + Drive
  - **Cout** : 5-15 EUR/user/mois

  **d) Microsoft 365 Sensitivity Labels**

  - **Inclus dans M365 E3 / E5**
  - Etiquetage automatique : Public / Interne / Confidentiel /
    Hautement Confidentiel
  - Chiffrement automatique selon etiquette
  - **AIP (Azure Information Protection)** : suite complete

  **e) Boxcryptor / Cryptomator** (chiffrement cloud)
  - Couche de chiffrement par-dessus OneDrive / Dropbox /
    Drive
  - Le cloud voit du contenu chiffre, toi du contenu clair
  - **Cryptomator** : OSS gratuit
  - **Boxcryptor** : 36 EUR/an

  **Le mot de passe : la pierre angulaire**

  Le chiffrement vaut son mot de passe. Regles :

  **1. Force minimale**
  - 14 caracteres minimum
  - Combinaison majuscules + minuscules + chiffres + symboles
  - Pas de mot du dictionnaire
  - **Phrase de passe** ideale : "Mon-3eme-Cousin-Mange-Du-Pain"

  **2. Transmission HORS-BANDE**
  - Le fichier va par email / cloud
  - Le mot de passe va par **autre canal** :
    - Telephone (le plus sur)
    - SMS (acceptable)
    - WhatsApp / Signal (mieux que SMS, chiffre bout-en-bout)
    - Pas dans le meme email !

  **3. Pas de mot de passe reutilise**
  - Chaque fichier sensible = mot de passe unique
  - Gestionnaire de mots de passe pour les retenir

  **4. Documentation pour deciffrement**
  - Si mot de passe perdu : fichier perdu
  - Documenter de facon securisee (vault, gestionnaire)
  - Politique entreprise : escrow possible (mot de passe en
    safe pour cas critique)

  **Comparaison des solutions :**

  | Solution | Cout | Niveau securite | Complexity |
  |---|---|---|---|
  | 7-Zip | Gratuit | Tres haut (AES-256) | Faible |
  | Office / LibreOffice MDP | Gratuit | Moyen-haut (selon version) | Tres faible |
  | PDF MDP | Gratuit | Faible | Tres faible |
  | ProtonMail | 5-15 EUR/mois | Tres haut | Faible |
  | Tresorit | 12-24 EUR/mois | Tres haut | Faible |
  | Sealit (FR) | 5-15 EUR/mois | Tres haut | Faible |
  | M365 Sensitivity Labels | Inclus E3 | Tres haut | Moyen |
  | Cryptomator | Gratuit | Tres haut | Moyen |

  **Recommandations selon contexte :**

  **Cas 1 : Envoi ponctuel un fichier sensible**
  - 7-Zip avec mot de passe + transmission MDP par telephone
  - Cout : 0 EUR

  **Cas 2 : Echanges reguliers avec un client/fournisseur**
  - ProtonMail Pro ou Sealit
  - Cout : 5-15 EUR/mois

  **Cas 3 : Equipe entiere echanges sensibles**
  - Microsoft 365 Sensitivity Labels (si deja M365)
  - Tresorit (si pas M365)

  **Cas 4 : Donnees ultra-sensibles**
  - Tresorit chiffre bout-en-bout + politiques DLP
  - Audit log complet
  - Authentification renforcee

  **Cas 5 : Stockage long-terme sensible**
  - Cryptomator par-dessus OneDrive / Drive
  - Ou Microsoft Sensitivity Labels avec chiffrement
    automatique

  **Le piege "PDF avec mot de passe c'est suffisant" :**

  PDF avec mot de passe d'ouverture (PDF 1.x) = chiffrement
  faible (40-128 bits). Cassable en quelques heures avec
  outils comme John the Ripper.

  Solution : chiffrer le PDF dans une archive 7-Zip a la place,
  ou utiliser PDF avec **chiffrement AES-256** (Adobe Acrobat
  Pro DC ou recent).

  **Le piege "j'envoie le mot de passe dans le meme email" :**

  Sense classique. Si l'email est intercepte (boite mail
  piratte, attaquant sur le reseau) = TOUT compromis.

  Solution : **canal different** pour le mot de passe.

  **Le piege "le destinataire n'a pas 7-Zip" :**

  - Sur Windows : **Windows Explorer** ouvre les ZIP nativement
  - Sur Mac : **Keka** (gratuit)
  - 7-Zip / WinRAR / PeaZip sur Windows
  - Le destinataire telecharge en 1 minute

  Si vraiment l'IT du client est restrictif : utiliser un
  service en ligne (ProtonMail) qui ne necessite pas de
  software cote destinataire.

  **Cas reels :**

  **Cas 1 - Cabinet d'avocats 2023 (donnees clients en clair)**

  Avocat envoie a son client le contrat (donnees sensibles +
  honoraires) en email Outlook standard. Le compte mail du
  client est piratte : tout le contrat exfiltre.

  Sanctions : 50 K EUR (CNIL) + procedure deontologique
  (Ordre des avocats).

  Cause : pas de chiffrement par defaut.

  **Cas 2 - PME 2022 (perte de mot de passe)**

  Comptable a chiffre une archive avec un mot de passe
  complexe. 6 mois plus tard, besoin de re-acceder. Mot de
  passe perdu. **Archive perdue**. Heureusement, sauvegarde
  off-site avec version anterieure non-chiffree.

  Lecon : politique de gestion des mots de passe + escrow.

  **Cas 3 - Hopital 2024 (RGPD compliance)**

  Hopital qui doit envoyer dossier patient a un autre. Standard
  : ProtonMail Pro (5 EUR/mois). Cout total : 60 EUR/an pour
  conformite GDPR + audit OK.

  **La regle d'or :**

  > **Donnees sensibles = chiffrement obligatoire avant envoi.
  > 7-Zip avec mot de passe HORS-BANDE = methode universelle
  > gratuite. Pour les volumes : ProtonMail, Tresorit, ou
  > Microsoft Sensitivity Labels. Mot de passe = jamais dans
  > le meme canal que le fichier.**
