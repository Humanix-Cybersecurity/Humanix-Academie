debrief: |
  Les **services de cloud storage** sont aujourd'hui omnipresents
  (Dropbox, Google Drive, OneDrive, iCloud, Box). Mais leurs
  usages doivent etre clairs pour eviter Shadow IT, fuites RGPD
  et perte de donnees.

  **Comparatif des services principaux :**

  | Service | Public | Forces | Faiblesses |
  |---|---|---|---|
  | **OneDrive Business** | Pro (M365) | Integration Office, 1-5 To, audit, MFA | Mobile parfois lent |
  | **Google Drive** (Workspace) | Pro (Google) | Collaboration temps reel, recherche IA | Search > storage |
  | **Dropbox Business** | Pro | Sync cross-platform robuste | Cher pour vol important |
  | **Tresorit** | Pro chiffre | Chiffrement bout-en-bout, GDPR native | Cher |
  | **Sealit / Whaller** (FR) | Pro FR | Hebergement FR, conformite | Communaute plus petite |
  | **iCloud** | Perso (Apple) | Integration native Apple | Pas pro-ready |
  | **Dropbox** (free) | Perso | 2 Go gratuit, simple | Pas pro |
  | **Drive** (free, Gmail) | Perso | 15 Go gratuit | Pas pro |

  **Regle 1 : Separation stricte perso / pro**

  **Pour le pro** :
  - Compte d'entreprise (OneDrive Business, Google Workspace,
    Dropbox Business)
  - SSO / MFA configure par le DSI
  - Acces depuis materiel pro
  - **Conformite** : audit log, retention, DLP

  **Pour le perso** :
  - Comptes perso (gratuits ou payants individuels)
  - Photos famille, documents prives
  - **JAMAIS** de donnees pro

  **Pourquoi cette separation est essentielle :**

  1. **RGPD** : donnees pro doivent etre sur outils valides
    par l'entreprise (article 32 - mesures techniques)
  2. **Propriete** : si tu pars, l'entreprise perd l'acces a
    tes donnees pro si elles sont chez toi
  3. **Securite** : le pro a souvent MFA + DLP, le perso
    rarement
  4. **Audit** : impossible d'auditer ce qui est sur ton perso

  **Regle 2 : Le bon outil pour le bon usage**

  **a) Documents Office (Word, Excel, PowerPoint)**
  - **Microsoft 365** : OneDrive / SharePoint = ideal
  - Coediting, version history, integration Office native

  **b) Documents Google (Docs, Sheets, Slides)**
  - **Google Workspace** : Drive = ideal
  - Recherche IA, collaboration tres fluide

  **c) Fichiers volumineux (videos, gros datasets)**
  - **OneDrive Business** : 1-5 To inclus
  - **Dropbox Business** : illimite sur Advanced
  - **Google Workspace Enterprise** : illimite

  **d) Donnees ultra-sensibles**
  - **Tresorit** : chiffrement bout-en-bout
  - **Microsoft 365 Sensitivity Labels** : etiquetage + chiffrement
  - **Cryptomator** par-dessus le cloud existant

  **e) Echange avec clients qui utilisent un autre cloud**
  - **Dropbox client** : Dropbox Business invitation OU upload
    sur OneDrive et partage de lien
  - **Drive client** : pareil
  - **Politique** : pas de creation de compte perso pour
    "depanner"

  **Regle 3 : Configuration de securite**

  **Pour le pro (OneDrive Business / Google Drive Workspace) :**

  - **MFA obligatoire** sur tous les comptes
  - **SSO** configure (Microsoft Entra ID / Google Identity)
  - **Audit log** active et consulte mensuellement
  - **DLP** (Data Loss Prevention) configure pour donnees
    sensibles
  - **Politique de partage** : nominatif par defaut, expiration
    30j (cf ep 03)
  - **Retention** : politique definie (RGPD)
  - **Backup** : verifier que c'est sauvegarde en plus

  **Pour le perso :**

  - MFA active (meme sur compte perso, par precaution)
  - Mot de passe fort + unique
  - Pas de partage public sauf vraiment public
  - Sauvegarde locale (cf saison sauvegardes)

  **Regle 4 : Politique d'entreprise claire**

  Une bonne politique d'usage definit :

  - **Outils valides** : OneDrive Business, Teams, SharePoint
    (M365 par exemple)
  - **Outils tolere** : selon contexte (Google Drive si
    travailler avec Google Workspace clients)
  - **Outils interdits** : usage perso pour donnees pro
  - **Procedure** pour demander un nouvel outil

  **Format** : 2-3 pages, simple, claire, dans l'intranet ou
  charte informatique.

  **Configuration recommandee Microsoft 365 :**

  Admin center > Settings > SharePoint > Sharing :

  - **External sharing** : New and existing guests
  - **Default link type** : Specific people (pas Anyone with
    link)
  - **Default link permission** : View
  - **Expiration** : 30 days
  - **Sensitivity labels** : actives pour Confidentiel /
    Hautement Confidentiel
  - **DLP** : detection donnees personnelles (NIR, IBAN, CB)

  **Configuration recommandee Google Workspace :**

  Admin > Apps > Drive and Docs > Sharing :

  - **Sharing outside organization** : ON
  - **Whitelist domains** : preferer si possible
  - **Default for link sharing** : Restricted
  - **Access checker** : Recipients only

  **Cas typiques :**

  **Cas 1 : "Mon client utilise Dropbox, je suis sur OneDrive"**

  Solutions :
  - **OneDrive partage externe** : invite le client, il accede
    sans creer compte Dropbox de ton cote
  - **Service tiers** : envoi via WeTransfer Pro, Smash, Tresorit
  - **Dropbox Business** : si echanges frequents, demander a
    l'entreprise

  **PAS** : creer un compte Dropbox perso pour stocker les
  documents client.

  **Cas 2 : "OneDrive est trop lent pour mes gros fichiers
  videos"**

  Solutions :
  - **Verifier la connexion** : VPN active ralentit OneDrive
  - **Microsoft 365 Stream** : pour videos, plus optimise
  - **Compression** : reduire la taille avant upload
  - **Demander au DSI** : si vraiment limitatif, peut-etre
    upgrade plan

  **PAS** : utiliser Dropbox perso pour les videos pro.

  **Cas 3 : "Je veux acceder a mes documents pro sur mon
  iPhone perso"**

  Solutions :
  - **App OneDrive / Google Drive sur iPhone** : connecte avec
    le compte pro, separation maintenue
  - **Microsoft Authenticator** : MFA depuis iPhone perso
  - **Politique BYOD** : si l'entreprise l'accepte, OK

  **PAS** : telecharger les fichiers pro dans iCloud perso.

  **Cas 4 : "Je veux partager une photo de l'evenement equipe
  avec tous"**

  Solutions :
  - **OneDrive partage interne** : tous dans l'organisation
  - **Microsoft Stream** : pour videos
  - **SharePoint site evenement** : centralise

  **PAS** : Drive perso "anyone with link".

  **Risques specifiques :**

  **Risque 1 : Sync automatique perso**

  - L'employe a Dropbox perso installe sur son PC pro
  - Il fait un copy-paste d'un fichier client dans Dropbox
    perso "par habitude"
  - Le fichier se sync automatiquement chez lui
  - Si compte Dropbox perso compromis = fuite

  Solution : ne pas installer Dropbox perso sur PC pro.

  **Risque 2 : Liens partages anciens**

  - Tu as partage un fichier en 2020 avec un client
  - Le client a quitte son entreprise en 2023
  - Le lien est encore actif
  - Tu n'as jamais fait audit

  Solution : audit annuel des partages externes.

  **Risque 3 : Compte perso reutilise pour pro**

  - Dropbox perso cree il y a 5 ans, mot de passe 'azerty'
  - L'employe a melange perso/pro
  - Compte compromis = fuite pro

  Solution : separation stricte + MFA partout.

  **Cas reels :**

  **Cas 1 - Cabinet conseil 2023 (perte de donnees)**

  Un consultant utilisait son Dropbox perso pour stocker les
  livrables clients. Il quitte l'entreprise. L'entreprise n'a
  pas l'acces aux 2 ans de livrables clients.

  Cout : 50 K EUR pour reprendre les dossiers + perte de
  confiance des clients.

  **Cas 2 - PME 2024 (fuite via sync auto)**

  Comptable avec Dropbox perso installe sur PC pro. Avait
  copie-colle un fichier paie dans Dropbox perso "par
  habitude". Compte perso compromis (mot de passe reutilise
  d'une autre fuite). Fichier paie expose.

  Sanctions : 30 K EUR (CNIL) + procedure interne.

  **Cas 3 - Startup tech 2024 (succes par politique claire)**

  DSI a defini une politique : OneDrive Business pour tout
  pro, Drive perso interdit sur PC pro. CASB detecte les
  installations non-conformes. Politique claire = 0 incident
  en 18 mois.

  **La regle d'or :**

  > **Separation stricte perso/pro. Le compte perso est pour
  > la vie privee, pas le travail. Pour echanges externes :
  > toujours via les outils pro (OneDrive partage externe,
  > services dedies). MFA + audit log + politique claire =
  > standard.**
