debrief: |
  Les **outils de collaboration** (Slack, Teams, Notion,
  Discord, Mattermost) sont devenus essentiels mais
  representent un risque majeur de fuites :

  - **Volumes** : milliers de messages
  - **Persistance** : tout reste (sauf retention configuree)
  - **Acces** large par defaut (canaux publics)
  - **Externes** invites
  - **Mobile / desktop** : capture facile

  Etudes Microsoft 2023 : 30% des breaches PME impliquent un
  outil de collaboration mal configure.

  **Slack par defaut : pieges classiques**

  **Piege 1 : Canaux publics**

  Quand tu crees un canal Slack, par **defaut** :
  - **Public** : tous les membres de l'organisation y ont
    acces
  - **Visible** dans la liste de canaux
  - **Recherche** dans tous les messages

  **Beaucoup d'employés** ne savent pas comment changer en
  prive (ou oublient).

  **Piege 2 : Invites externes (guests)**

  Slack permet d'inviter des consultants / clients comme
  "guests". Mais :
  - **Acces** souvent etendu par defaut
  - **Permissions** mal granulaires
  - **Oubli** de revoquer après mission

  **Piege 3 : Apps tiers (integrations)**

  Slack a des centaines d'apps (Bots) :
  - **GitHub bot** lit les messages
  - **Jira bot** lit les messages
  - **Apps tierces** lisent les messages
  - **Souvent** : permissions excessives

  **Piege 4 : DM (messages directs)**

  Les DM sont **prives entre 2 personnes** mais :
  - **Admins workspace** peuvent les exporter (cas Slack
    Pro/Business+)
  - **Recherche** par admin possible
  - **Pas vraiment confidentiel**

  **Piege 5 : Historique infini**

  - Slack Free : 90 jours d'historique
  - Slack Pro : illimite
  - Tout reste = base de données attaquable

  **Configuration recommandee :**

  **Niveau 1 : Acces et rôles**

  - **Single Sign-On (SSO)** : authentification entreprise
  - **MFA obligatoire**
  - **Roles bien definis** : Admin / Member / Guest /
    Single-channel guest

  **Niveau 2 : Politique de canaux**

  - **Canaux clients = PRIVES par defaut**
  - **Canaux stratégiques** = prives (DG, COMEX)
  - **Canaux RH / juridique** = prives
  - **Canaux généraux / annonces** = publics OK
  - **Canaux 'Random'** = OK

  **Niveau 3 : Gestion des invites externes**

  - **Single-channel guest** : acces a 1 canal seulement
  - **Multi-channel guest** : acces spécifique (éviter
    'all channels')
  - **Date d'expiration** automatique (30-90 jours)
  - **Revue trimestrielle** : qui sont les externes actifs ?

  **Niveau 4 : Apps tiers**

  - **Liste blanche** d'apps approuvees par RSSI
  - **Permissions minimales** : just enough
  - **Audit** annuel des apps

  **Niveau 5 : Retention**

  - **Configuration retention** : 30/90 jours pour la plupart
    des canaux (sauf juridique)
  - **Conservation legale** : 5-10 ans selon obligation
  - **Suppression auto** = moins de risque historique

  **Niveau 6 : DLP (Data Loss Prévention)**

  Microsoft 365 / Slack Enterprise :
  - **Détection** patterns sensibles (NIR, IBAN, CB)
  - **Alerte** ou **bloquage** automatique
  - **Audit log**

  **Configuration Slack workspace recommandee :**

  Admin Settings > Workspace :

  - **Default channel** : restrict création publique aux
    admins
  - **External sharing** : restrict
  - **App approval** : requis
  - **Retention policy** : 90 jours par defaut (override pour
    canaux spécifiques)

  **Les 8 mauvaises pratiques courantes :**

  **1. "Tout le monde est dans le canal"**
  - Acces large par defaut
  - Solution : prive sauf si justifie public

  **2. "Documents partages dans le canal"**
  - Slack n'est pas un cloud storage
  - Solution : SharePoint / Drive avec partage contrôle

  **3. "DM 'prive' avec mots de passe"**
  - DM pas vraiment prives (admin peut acceder)
  - Solution : gestionnaire mots de passe + canaux secures

  **4. "Invites externes 'pour 1 mission' jamais retires"**
  - Comptes fantomes
  - Solution : expiration automatique

  **5. "Slack notifications avec contenu sur écran de
  verrouillage"**
  - Mots de passe / info sensible visibles
  - Solution : notifications sans contenu

  **6. "Connecte sur PC perso / public"**
  - Données pro sur app perso
  - Solution : usage strict pro / pro

  **7. "Captures / forward sans précaution"**
  - Slack screenshots exposes / shared
  - Solution : sensibilisation + DLP

  **8. "Canaux avec noms revelateurs"**
  - "#acquisition-acme-2024" devine
  - Solution : codes (ex: "#projet-X")

  **Cas reels :**

  **Cas 1 - PME 2023 (canal Slack public + ex-employé)**

  Équipe projet client a #project-acme avec 80 membres
  (tout le monde par defaut Slack public). Un ex-employé en
  conflit (vue de la suite avec succes) a exfiltre toute
  l'historique avant de partir.

  Cout :
  - **NDA viole** = procédure client
  - **Information vendue** au concurrent (suppose)
  - **15 K EUR** indemnisation client
  - **Audit complet** Slack obligatoire

  **Cas 2 - Editeur SaaS 2024 (DM compromis)**

  Employe pense ses DM Slack proteges. Recoit un mot de
  passe d'un collègue ("le SSH du serveur prod c'est
  XXXXX"). Compte du collègue compromis -> admin Slack
  exfiltre tous ses DMs -> mots de passe extraits.

  Breach prod 2 semaines après.

  **Cas 3 - Cabinet conseil 2022 (consultant externe)**

  Consultant externe ajoute a Slack pour 3 mois. Acces
  multi-channel guest. Mission terminee, consultant
  toujours dans Slack. 6 mois après : consultant rejoint
  concurrent + Slack toujours accessible.

  Cout : information cliente vendue. Sanction : -50 K EUR.

  **Cas 4 - PME 2024 (apps tierces excessives)**

  Audit DSI : 75 apps tierces installees dans Slack
  workspace. Beaucoup avec permissions "lire tous les
  messages". Suspicion (non prouvee) que certaines
  exfiltraient.

  Action :
  - **Liste blanche** d'apps approuvees (15)
  - **Suppression** des autres
  - **Politique** d'approbation pour nouvelles apps

  **Microsoft Teams - configurations similaires :**

  **Équipes / Canaux** :
  - **Équipe privée** = canaux non visibles publiquement
  - **Canal prive** dans équipe publique = acces restreint
  - **Canal partage** : multi-tenant

  **Invites** :
  - **External access** (federation) : autres organisations
  - **Guest access** : externe ajoute a une équipe
  - **Permissions** configurables

  **Notion - cas particulier :**

  Notion a des permissions complexes :
  - **Shared with everyone** : tout le workspace
  - **Public to internet** : DANGEREUX (cf ep 03)
  - **Specific people** : recommande

  Audit regulier des pages "Public to internet" obligatoire.

  **La règle d'or :**

  > **Outils de collaboration = configuration prive par
  > defaut + invites externes encadres + DLP + retention
  > limitee + audit annuel apps tierces. Canaux clients =
  > prives. Mots de passe = pas dans Slack/Teams. Captures
  > sensibilises. Configuration workspace par RSSI/DSI.**
