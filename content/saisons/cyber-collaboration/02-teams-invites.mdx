debrief: |
  Microsoft Teams permet plusieurs **modes de collaboration
  externe**. Choisir le bon = securiser la collaboration sans
  exposer l'organisation.

  **Les 5 modes de collaboration externe Teams :**

  **Mode 1 : Guest Access (le plus utilise)**

  - Externe ajoute comme **invite** dans l'equipe
  - **Acces** : equipe et ses canaux (selon permissions)
  - **Pas de licence** Microsoft 365 (gratuit)
  - **Authentification** : compte Microsoft externe (peut
    etre cree gratuitement)
  - **Pour qui** : consultants, clients, fournisseurs

  Permissions configurables :
  - Pas creation de canaux
  - Pas suppression de fichiers
  - Pas ajout d'invites supplementaires

  **Mode 2 : External Access (Federation)**

  - **Federation** entre 2 organisations Microsoft 365
  - L'externe utilise SON propre tenant
  - **Communication** chat / appel possible
  - **Pas d'acces** a tes equipes / fichiers

  Pour qui : partenaires reguliers avec leur propre Teams.

  **Mode 3 : Shared channels (Microsoft 365 Connect)**

  - **Canal partage** entre organisations
  - **Externe** voit ce canal mais reste dans son tenant
  - **Plus securise** que Guest Access (acces tres limite)
  - **Necessite** : tous parties sur Microsoft 365

  Disponible depuis 2022, recommande pour collaboration B2B.

  **Mode 4 : Anonymous / lien d'invitation**

  - **Lien** envoye, n'importe qui peut rejoindre
  - **DANGEREUX** : pas de controle d'identite
  - **Eviter** sauf reunions tres ouvertes (webinaires)

  **Mode 5 : Compte M365 invite (rare)**

  - **Licence M365** payante pour l'externe
  - **Acces** complet comme employe
  - **Cher** et **excessif** dans la plupart des cas

  Pour qui : consultants long-terme integres dans l'equipe.

  **Choisir le bon mode :**

  | Cas | Mode recommande |
  |---|---|
  | **Client suivi projet 3 mois** | Guest Access (equipe privee) |
  | **Consultant SAP 6 mois** | Guest Access ou compte M365 |
  | **Fournisseur ponctuel** | Shared Channel ou Guest Access (limite) |
  | **Partenaire avec Teams** | External Access (federation) |
  | **Webinaire public** | Anonymous (lien public) |

  **Procedure d'invitation securisee :**

  **Etape 1 : Equipe dediee**

  Pour chaque projet externe :
  - **Cree une equipe** specifique
  - **Equipe privee** (pas publique a l'organisation)
  - **Pas la 'Direction generale' / equipe globale**

  **Etape 2 : Permissions externe**

  - **Member** : participation normale
  - **Owner** : pas pour externes (sauf cas specifique)
  - **Guest** : permissions reduites par defaut

  Reglages typiques pour Guest :
  - **Cannot create channels**
  - **Cannot delete channels**
  - **Cannot add/remove apps**
  - **Cannot delete files**

  **Etape 3 : Onboarding cyber externe**

  Avant d'envoyer l'invitation :
  - **NDA** signe (Non-Disclosure Agreement)
  - **Charte cyber** lue
  - **Liste de canaux** auxquels il aura acces
  - **Date d'expiration** definie

  **Etape 4 : Invitation nominative**

  - **Email externe** valide
  - **Pas de lien generique**
  - **Compte Microsoft** que l'externe utilise

  **Etape 5 : Configuration retention / DLP**

  - **Retention policy** : selon politique
  - **DLP** : protection donnees sensibles
  - **Sensitivity labels** : appliques aux fichiers

  **Etape 6 : Suivi pendant la mission**

  - **Activite** auditable (logs)
  - **Comportement** anormal a surveiller
  - **Mise a jour** des acces si besoin

  **Etape 7 : Offboarding (cf saison cyber-rh ep 02 et
  supply-chain ep 06)**

  Apres mission :
  - **Revocation** du Guest Access (jour J)
  - **Suppression** apres 30 jours
  - **Audit** : aucun acces actif residual
  - **Documentation**

  **Configuration Microsoft 365 admin :**

  Microsoft 365 admin center > Teams > Org-wide settings :

  - **Guest access** : ON (mais avec restrictions)
  - **External access** : selectif (whitelist domaines)
  - **Anonymous access** : restrictif

  Microsoft 365 admin > Microsoft Entra ID > External
  Identities :
  - **Guest user permissions** : restrictives par defaut
  - **External collaboration restrictions**

  **Les 6 erreurs frequentes :**

  **Erreur 1 : Externe dans equipe globale**

  - Voit tous les canaux internes
  - Strategies, RH, finances visibles

  Solution : equipe dediee.

  **Erreur 2 : Pas de date d'expiration**

  - Externe avec acces 5 ans apres mission
  - Compte fantome

  Solution : expiration automatique + revue.

  **Erreur 3 : NDA pas signe**

  - Pas de protection juridique
  - Fuite difficile a contrer legalement

  Solution : NDA prealable systematique.

  **Erreur 4 : Permissions excessives**

  - Externe peut creer canaux / supprimer fichiers
  - Risque sabotage / fuite

  Solution : permissions minimales (Guest restrictif).

  **Erreur 5 : Pas d'audit**

  - Activite non surveillee
  - Comportements anormaux passent inaperçus

  Solution : audit log periodique.

  **Erreur 6 : Offboarding oublie**

  - Compte fantome apres mission
  - Acces persistent

  Solution : offboarding programme et trace.

  **Cas reels :**

  **Cas 1 - PME 2024 (offboarding rate)**

  Consultant SAP ajoute a Teams pour 6 mois (Guest Access).
  Mission terminee. Pas de revocation. 12 mois apres : audit
  DSI revele guest toujours actif + connexion mensuelle.

  Verification : consultant a vu informations confidentielles
  pendant 12 mois sans probleme business (mais risque).

  Action :
  - Revocation immediate
  - Audit complet acces
  - Politique offboarding renforcee

  **Cas 2 - Cabinet 2023 (NDA absent)**

  Stagiaire 6 mois ajoute a Teams sans NDA. Apres stage :
  expression d'interet pour concurrent. NDA absent =
  protection juridique nulle.

  Couts : 30K EUR de business potentiellement compromise.

  **Cas 3 - PME 2022 (permissions excessives)**

  Externe (consultant) ajoute comme "Owner" par erreur. A
  modifie / supprime des fichiers + cree nouveaux canaux.
  Decouverte : 2 mois de degats accumulees.

  Solution post-incident : politique stricte d'attribution
  des roles + revues mensuelles.

  **Cas 4 - PME 2024 (succes Shared Channels)**

  PME utilise Shared Channels pour partenaire. Acces
  partenaire limite a 1 canal (echange documents seulement).
  Pas d'acces aux autres equipes ni discussions internes.

  Resultat : collaboration efficace + securite preservee.

  **Pour le RSSI :**

  Politique de gestion des externes :

  - **Categorie A - Stratégique** : NDA renforce + audit
    bi-mensuel + Shared Channels
  - **Categorie B - Standard** : NDA standard + Guest
    Access + offboarding clair
  - **Categorie C - Ponctuel** : External Access ou Anonymous

  Indicateurs :
  - Nombre d'externes actifs / mois
  - Temps moyen d'offboarding (cible : < 7 jours)
  - Audit acces externes (trimestriel)

  **La regle d'or :**

  > **Externes dans Teams : equipe dediee privee + Guest
  > Access avec permissions minimales + NDA signe + date
  > expiration + offboarding programme + audit. Eviter
  > 'tout l'organisation' + 'sans expiration'. Shared
  > Channels = standard moderne pour partenaires Microsoft
  > 365.**
