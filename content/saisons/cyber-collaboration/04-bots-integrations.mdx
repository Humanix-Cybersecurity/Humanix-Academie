debrief: |
  Les **bots / integrations / apps tierces** dans Slack /
  Teams / Notion sont une source de fuites majeure souvent
  ignoree.

  Etudes CrowdStrike 2023 : 60% des PME ont plus de 50 apps
  tierces installees dans leur workspace, sans politique
  d'approbation.

  **Pourquoi c'est risque :**

  **Risque 1 : Permissions excessives**

  La plupart des apps demandent par defaut :
  - **Lire tous les messages** (sur Slack)
  - **Lire tous les utilisateurs**
  - **Lire les fichiers**
  - **Ecrire dans les canaux**

  Souvent **plus que necessaire** pour leur fonction reelle.

  **Risque 2 : Securite chez le tiers**

  Si l'app provider est compromis (breach):
  - **Toutes les donnees** que l'app a lu sont compromises
  - **Token API** vole = acces a TON workspace par
    l'attaquant

  **Cas reel MOVEit 2023** : faille dans MOVEit (logiciel
  transfert de fichiers) = compromission de centaines
  d'organisations.

  **Risque 3 : Insider chez le provider**

  Employe rancunier de l'app provider peut :
  - Acceder aux donnees clients
  - Exfiltrer
  - Vendre

  **Risque 4 : RGPD**

  Chaque app tierce = sous-traitant RGPD :
  - **Article 28** : contrat de sous-traitance obligatoire
  - **Registre traitements** : declarer
  - **Hebergement** : verifier (UE / Cloud Act)
  - **Securite** : verifier conformite

  Souvent **non-respecte** dans les PME.

  **Risque 5 : Apps zombie (oubliees)**

  - Installee il y a 3 ans pour un projet
  - Plus utilisee
  - **Acces toujours actif**
  - **Compte fantome** equivalent

  **Procedure d'audit :**

  **Etape 1 : Inventaire complet**

  **Slack** :
  - Workspace settings > Apps
  - Liste de toutes les apps installees
  - Pour chaque : qui l'a installee, quelles permissions

  **Microsoft Teams** :
  - Admin center > Teams apps
  - Manage apps > Apps and policies
  - Custom apps + Microsoft apps + 3rd-party

  **Notion** :
  - Settings > Connected accounts / Integrations

  **Etape 2 : Categorisation**

  Pour chaque app :

  | Critere | Question |
  |---|---|
  | **Utilite metier** | Activement utilisee ? |
  | **Permissions** | Excessives ? Justifiees ? |
  | **Provider** | Reputation ? Conformite ? |
  | **RGPD** | Contrat sous-traitance ? Registre ? |
  | **Hebergement** | UE / hors-UE ? Cloud Act ? |
  | **Securite** | Audit / certif (SOC 2, ISO 27001) ? |
  | **Activite** | Derniere utilisation ? |

  Categories :
  - **Approuvee + utilisee** : keep
  - **Approuvee + non-utilisee** : suppress (compte fantome)
  - **Non-approuvee** : evaluation
  - **Risquee** : suppression immediate

  **Etape 3 : Liste blanche**

  Apres audit, etablir liste **approuvee** :
  - **Categorie** (collaboration, analytics, dev, etc.)
  - **Permissions** minimales
  - **Documentation** justification
  - **Renouvellement** annuel

  **Etape 4 : Suppression apps non-essentielles**

  - **Notification** equipes utilisateurs
  - **Migration** vers alternatives si necessaire
  - **Suppression** apps obsoletes
  - **Documentation** : pourquoi suppression

  **Etape 5 : Politique d'approbation**

  Pour nouvelles apps :
  - **Demande formelle** au RSSI / DSI
  - **Justification** metier
  - **Verification** RGPD + securite
  - **Permissions** minimales
  - **Approbation** ou refus motive

  **Apps Slack populaires - guide rapide :**

  **Faible risque (often OK)** :
  - **Polly / Polldaddy** : sondages
  - **Giphy** : GIFs (usage social, pas donnees critiques)
  - **Calendly** : planification (acces calendrier seulement)

  **Moyen risque (verification)** :
  - **GitHub bot** : acces code source (necessite contrat
    sous-traitance + RGPD)
  - **Jira / Trello** : gestion tickets
  - **Hubspot / Salesforce** : CRM (donnees clients)

  **Risque eleve (audit minutieux)** :
  - **Gmelius / Front** : email (lit beaucoup de messages)
  - **Zapier / IFTTT** : automatisation (acces multi-app)
  - **OpenAI ChatGPT integration** : envoi a OpenAI

  **Risque tres eleve (eviter par defaut)** :
  - **Apps inconnues / obscures** : potentielles malveillantes
  - **Apps avec permissions tres larges** sans justification

  **Configuration Slack admin recommandee :**

  Workspace settings > Permissions > Apps :

  - **App approval** : require approval for installs
  - **App management** : restrict to admins
  - **Workspace owners** : peuvent valider les nouvelles
    apps
  - **App directory** : limited to approved

  **Configuration Microsoft Teams admin :**

  Teams admin center > Teams apps > Permission policies :

  - **Default policy** : restrictive (Microsoft apps + custom
    only, no 3rd-party)
  - **Custom policy** : pour utilisateurs specifiques

  **Le piege "c'est gere par mon equipe IT" :**

  Souvent FAUX. Beaucoup d'employes installent eux-memes
  des apps Slack / Teams (Self-Service). DSI ne sait meme pas.

  Solution : politique d'approbation centrale + audit
  periodique.

  **Le piege "l'app est gratuite, donc safe" :**

  Faux. Apps gratuites :
  - **Modele economique** : monetisation des donnees ?
  - **Permissions** parfois excessives
  - **Hebergement** : pas verifie

  "If you're not paying for the product, you ARE the
  product."

  **Le piege "App officielle Slack/Microsoft" :**

  Pas tous les apps "Slack apps" ou "Teams apps" sont
  developpees par Slack/Microsoft :
  - **Marketplace** : apps tierces validees mais pas
    auditees
  - **Validation Slack/Microsoft** : technique, pas
    securite/RGPD complete

  Audit interne necessaire.

  **Cas reels :**

  **Cas 1 - PME 2023 (75 apps audit)**

  PME 80 personnes : 75 apps Slack installees. Audit DSI :
  - 30 utilisees activement
  - 25 utilisees occasionnellement
  - 20 non utilisees depuis > 6 mois (zombies)
  - 5 avec permissions excessives non justifiees

  Action :
  - Suppression de 45 (zombies + excessives)
  - Liste blanche de 30 (RSSI-approved)
  - Politique d'approbation
  - Audit annuel

  Resultat : -60% surface attaque + 100% RGPD compliance.

  **Cas 2 - Cabinet conseil 2022 (MOVEit breach)**

  Cabinet utilisait MOVEit pour transferts. Faille MOVEit
  2023 (CVE-2023-34362) = compromission massive.

  Cabinet : 5 mois de donnees clients exfiltrees + 200 K EUR
  incident response + reputation degradee.

  Lecon : audit reguliere des apps + monitoring breaches
  fournisseurs.

  **Cas 3 - Startup 2024 (OpenAI integration)**

  Startup avait integration ChatGPT dans Slack pour aide aux
  employes. Conversations avec ChatGPT envoyees a OpenAI
  (US) sans :
  - Contrat sous-traitance RGPD
  - Information employes
  - Registre traitement

  Audit interne RGPD : non-conformite. Action : politique
  d'usage IA + contrat OpenAI Enterprise + information.

  **Cas 4 - PME 2022 (insider Slack provider)**

  App tierce installee pour analytics. Provider eu un
  insider qui a vendu des access tokens d'environ 100
  workspaces.

  PME victime : compromission acces Slack pendant 2
  semaines. Decouverte par audit logs.

  Action : suppression provider + nouveau choix + audit
  approfondi.

  **Apps "AI agents" et MCP (Model Context Protocol) :**

  Tendance 2024-2026 : agents IA qui se connectent
  directement a Slack / Teams via MCP.

  Risques :
  - **Acces large** par defaut
  - **Donnees envoyees** a fournisseur IA (US souvent)
  - **Comportement** parfois imprevisible
  - **RGPD** complexe

  Recommandation :
  - **Apps MCP / IA** = approbation specifique RSSI
  - **Donnees** classees avant integration
  - **Sensitivity labels** appliquees
  - **Audit** intensif

  **Pour le RSSI - dashboard recommande :**

  KPI mensuels :
  - **Nombre d'apps actives** (cible : -10% par an si
    excessif)
  - **Apps non utilisees > 90 jours** (cible : 0)
  - **Apps avec permissions excessives** (cible : 0)
  - **Apps non-RGPD-compliant** (cible : 0)
  - **Demandes d'approbation** (volume + acceptees / refusees)

  **La regle d'or :**

  > **Apps tierces = surface d'attaque souvent oubliee. Audit
  > complet (60% des PME ont 50+ apps). Liste blanche
  > RSSI-approved. Permissions minimales. Suppression apps
  > zombies + excessives. Politique d'approbation pour
  > nouvelles. RGPD : contrat sous-traitance + registre. Audit
  > annuel.**
